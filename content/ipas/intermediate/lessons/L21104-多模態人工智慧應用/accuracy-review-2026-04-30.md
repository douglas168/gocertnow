# Accuracy Review — L21104: 多模態人工智慧應用
Date: 2026-04-30
Reviewers: Claude (independent) + Gemini (adversarial)

---

## Summary

The guide is structurally sound — the Big Picture flow, decision trees, and Trap Clinic format are effective. Core concepts (CLIP dual-encoder, cross-attention Q/K/V direction, Flamingo architecture) are largely correct. However, three classes of errors require fixes before publishing:

1. A misleading taxonomy decision (CLIP classified as Late Fusion)
2. Version-blind claims about Stable Diffusion's text encoder
3. DALL-E 3 training objective stated incorrectly
4. LLaVA architecture claim outdated (linear vs. MLP)
5. GPT-4o "unified token space" stated as fact when it is OpenAI inference/speculation

Convergence: Claude and Gemini independently flagged items 1–4. Item 5 flagged by both with slight difference in framing.

## Critical Errors (must fix before publishing)

### Error 1: CLIP classified as "晚期融合 (Late Fusion)" in inference
- **Location**: Section 4.2, Table 4 (Late Fusion row, "代表: CLIP 推論"), Section 6 Q6 answer, Table in Section 7
- **Claim**: "CLIP 在推論階段屬於晚期融合 (Late Fusion)…各模態各自跑獨立模型，只在輸出層合流"
- **Error**: Late Fusion (晚期融合) means decision-level fusion — independent classifiers each produce class predictions (logits/probabilities) that are then voted or weighted. CLIP does not output class predictions from independent classifiers; it computes dot-product similarity between embeddings in a shared metric space. This is Dual-Encoder / metric-learning alignment, not decision-level voting. On an IPAS exam, a question may present CLIP (embedding similarity) and a voting-ensemble (decision fusion) as two distinct options — classifying both as "Late Fusion" will cause incorrect answers.
- **Fix**: Remove CLIP from the Late Fusion representative column. In Section 4.2, add a note: "CLIP 的推論機制（雙編碼器相似度比對）嚴格上是「特徵對齊 / 度量學習」，在部分文獻中歸類為晚期融合的廣義形態，但考試中 Late Fusion 代表的是決策層投票（logits 加權）。CLIP 的正確考試標籤是「雙編碼器對比學習（Dual Encoder Contrastive）」，不要用晚期融合代替。"

### Error 2: Stable Diffusion text encoder stated as "CLIP ViT-L/14" without version qualification
- **Location**: Section 5 Exam Rule ("看到「SD 不用 CLIP」→ 錯！SD 的文字編碼器就是 CLIP ViT-L/14"), Section 6 table, Section 7 table, Trap 6, Q22
- **Claim**: "SD 的文字編碼器就是 CLIP ViT-L/14"
- **Error**: Only SD 1.x uses CLIP ViT-L/14 (OpenAI). SD 2.x uses OpenCLIP ViT-H/14. SDXL uses dual encoders (CLIP ViT-L/14 + OpenCLIP ViT-bigG/14). SD 3.x and Flux use T5-XXL plus CLIP encoders. The unqualified claim is false for any SD version after 1.5. More importantly, SD 3/Flux (T5-based conditioning) is likely to appear in 2026 exam contexts given its industry dominance.
- **Fix**: Replace with: "SD 1.x 的文字編碼器是 CLIP ViT-L/14；SD 2.x 換為 OpenCLIP ViT-H/14；SDXL 用雙編碼器；SD 3.x / Flux 加入 T5-XXL。考試若不指定版本，預設 SD 1.5 = CLIP ViT-L/14，但要知道新版本的差異。"

### Error 3: DALL-E 3 training objective stated incorrectly
- **Location**: Section 7 Table ("DALL-E 2/3 | 訓練目標: Diffusion MSE + CLIP alignment")
- **Claim**: DALL-E 3 uses "Diffusion MSE + CLIP alignment"
- **Error**: DALL-E 3's distinguishing innovation is training with high-quality synthetic captions generated by GPT-4V, not CLIP-alignment loss. DALL-E 3 uses a latent diffusion model with T5-based text conditioning. Its alignment improvement comes from recaptioning the training dataset, not from a CLIP alignment loss during diffusion. The "CLIP alignment" description applies to DALL-E 2's prior stage.
- **Fix**: Separate DALL-E 2 and DALL-E 3 rows. DALL-E 2: CLIP prior + diffusion decoder. DALL-E 3: Latent diffusion + T5 conditioning + synthetic recaptioning (GPT-4V). Remove "DALL-E 3" from the CLIP alignment claim.

### Error 4: LLaVA described as using "線性投影" — outdated for all practical versions
- **Location**: Section 7 Table ("LLaVA | 線性投影 Prefix（非交叉注意力）"), Trap 7, Q13
- **Claim**: "LLaVA 使用線性投影 + prefix 拼接"
- **Error**: LLaVA 1.0 (original) used a single linear layer. LLaVA 1.5 (the industry baseline, widely cited in 2024–2025) upgraded to a 2-layer MLP with GELU activation. LLaVA-NeXT / LLaVA 1.6 uses MLP + AnyRes for high-resolution. Calling it "linear projection" is technically valid only for the original paper (2023) and misleads students about the current state of the architecture.
- **Fix**: Update to "MLP 投影（LLaVA 1.5+）或線性投影（LLaVA 1.0 原版）+ prefix 拼接；核心特點：非交叉注意力。" Keep the key exam point (not cross-attention) intact.

### Error 5: GPT-4o "統一 token 空間" stated as confirmed fact
- **Location**: Section 7 Table, Section 9 Trap 9, Final Oral Recall item 7
- **Claim**: "原生多模態（統一 token 空間）"
- **Error**: OpenAI confirmed GPT-4o is end-to-end trained from scratch across modalities ("Omni" = all modalities). However, "unified token space" (i.e., that all modalities share the same discrete/continuous token vocabulary) is technical speculation — OpenAI has not published architectural details confirming this. On an IPAS exam the safe term is "原生多模態 (Native Multimodal)" or "端到端多模態訓練 (End-to-End Multimodal Training)." Using "統一 token 空間" could be wrong if OpenAI uses modality-specific encoders that feed into a joint transformer, which is a distinct architecture.
- **Fix**: Replace "統一 token 空間" with "端到端原生多模態訓練" and add a note: "(OpenAI 未公開完整架構，統一 token 空間是合理推論但非官方確認)"

## Minor Issues

1. **Section 1 — Flamingo listed under "晚期融合 Late Fusion" in the process diagram**: The big picture diagram lists "晚期融合 Late Fusion → 各自模型，輸出層合流（CLIP 推論）" which further entrenches the CLIP-as-Late-Fusion error.

2. **Section 5 — "對稱 InfoNCE" terminology**: The CLIP paper does not use the term "symmetric InfoNCE." It uses a cross-entropy loss in both directions. "Symmetric" (雙向對稱) is correct in spirit but may cause confusion on an exam that uses the paper's exact terminology. Safer: "雙向對稱對比損失 (bidirectional symmetric contrastive loss)"。

3. **Section 7 — Whisper described as doing "語音活動偵測 (VAD)"**: Whisper does detect voice activity implicitly, but it is not the standard choice for VAD — dedicated models (Silero VAD, WebRTC VAD) are used in production pipelines. This is a minor overstatement.

4. **Section 7 — Gemini described as "預訓練即多模態"**: Gemini 1.0–1.5 series were pre-trained as multimodal from scratch. However, Gemini Ultra vs Pro vs Flash have different architectures and capabilities. This is slightly oversimplified but not exam-critical.

## Missing Key Concepts

1. **VQ-VAE / discrete image tokenization**: Critical for understanding how autoregressive multimodal models (Chameleon, some Gemini variants) represent images as tokens. Likely to appear in 2026 exams given the shift from diffusion to AR generation.

2. **AnyRes / dynamic resolution (LLaVA-NeXT, InternVL)**: High-resolution OCR and multi-page document understanding are IPAS practical application scenarios in Taiwan context.

3. **Modality Gap**: CLIP embeddings have a known "modality gap" (image and text embeddings cluster in separate regions of the shared space). This is examined at intermediate level.

4. **In-context multimodal few-shot**: How Flamingo and GPT-4o handle interleaved image-text sequences. The guide mentions Flamingo's few-shot capability but doesn't explain the mechanism.

## Verdict

**Grade: B**

Core architecture logic is correct: cross-attention Q/K/V assignments, CLIP training objective, Flamingo's Gated Cross-Attention + Perceiver Resampler, Whisper's encoder-decoder structure. The decision trees and trap clinic are pedagogically excellent.

However, the CLIP-as-Late-Fusion taxonomy error is dangerous (will produce wrong exam answers), and three named-model facts are outdated or wrong (SD encoder versions, DALL-E 3 training objective, LLaVA projection type). Fix the 5 critical errors before publishing; the remaining items are minor refinements.

**Publish-blocking: 5 critical errors. Do not publish without fixing Critical Errors 1–4 at minimum.**
