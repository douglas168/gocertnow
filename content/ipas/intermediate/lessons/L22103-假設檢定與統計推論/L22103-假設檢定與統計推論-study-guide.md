## 1. Big Picture / Core Pipeline 🔥🔥🔥

### 先懂一句話

`假設檢定（Hypothesis Testing）` 是用樣本資料，判斷「某個差異、效果或關聯」是否有足夠證據支持。它不是證明真理，而是在不確定中做有規則的決策。

### Everyday Analogy

像公司要不要採用新版流程。你不會只看一兩個人的感覺就下結論，而是先假設「新版沒有差」，再看資料是否明顯到足以推翻這個保守說法。

### 在整體流程中的位置

```text
商業 / 研究問題
→ 寫出 H0 / H1
→ 設定 α
→ 選擇 t / χ² / F 檢定
→ 算出檢定統計量與 p-value
→ 比較 p-value 與 α
→ 拒絕 H0 或無法拒絕 H0
```

### Key Concepts

| 題目問的是 | 想到 |
|---|---|
| 是否有差異、有效果、有關聯 | 先寫 `H0 / H1` |
| `p-value = 0.03`、`α = 0.05` | 比較 `p-value` 與 `α` |
| `95% CI` 是否包含某假設值 | 用信賴區間對照檢定結論 |
| 平均數比較 | `t-test` 或 `F-test / ANOVA` |
| 類別變數是否獨立 | `χ² test of independence` |

本課邊界要記清楚：

- 會考：假設檢定框架、`α / β / p-value / CI`、`t-test`、`χ² test`、`F-test / one-way ANOVA`。
- 不展開：非參數檢定、效果量、貝氏推論、迴歸、機器學習專屬統計、多因子 ANOVA 與事後檢定。

### Exam Rule

```text
p-value ≤ α → 拒絕 H0
p-value > α → 無法拒絕 H0
平均數比較 → t-test 或 F-test
類別 x 類別 / 是否獨立 → χ² test
三組以上平均數 → F-test / one-way ANOVA
```

### Quick Check

題目給 `α = 0.05`、`p-value = 0.08`，正確結論是什麼？

答案：無法拒絕 `H0`。因為 `0.08 > 0.05`，資料證據還不夠強。

## 2. 假設檢定框架：H0 / H1 / 單尾雙尾 🔥🔥🔥

### 先懂一句話

`H0` 是保守起點，通常代表「沒有差異、沒有效果、沒有關聯」。`H1` 是你想檢查的替代說法，通常代表「有差異、有效果、有關聯」。

### Everyday Analogy

像法庭中的「先假設無罪」。除非證據夠強，否則不會輕易推翻原本的保守立場。

### 在整體流程中的位置

```text
題目情境 → 寫 H0 / H1 → 選 α 與檢定方法 → 看 p-value → 下結論
```

### Key Concepts

| 名稱 | 常見意思 | 題目常見句型 |
|---|---|---|
| `H0` 虛無假說（Null Hypothesis） | 無差異、無效果、無關聯 | 平均數相同、兩變數獨立 |
| `H1` 對立假說（Alternative Hypothesis） | 有差異、有效果、有關聯 | 平均數不同、兩變數不獨立 |

例子：

| 情境 | `H0` | `H1` |
|---|---|---|
| 新版派單流程是否改變平均送達時間 | 新舊平均送達時間相同 | 新舊平均送達時間不同 |
| 會員等級與回購是否有關 | 會員等級與回購彼此獨立 | 會員等級與回購不獨立 |

單尾與雙尾的差別在於 `H1` 有沒有指定方向：

```text
雙尾檢定：H1: μ ≠ μ0   看是否不同，不管變大或變小
右尾檢定：H1: μ > μ0   看是否變大 / 提升
左尾檢定：H1: μ < μ0   看是否變小 / 降低
```

```text
題目只問「是否不同」 → 雙尾
題目明確問「是否提升」 → 右尾
題目明確問「是否降低」 → 左尾
```

### Exam Rule

```text
無差異 / 無效果 / 無關聯 → H0
有差異 / 有效果 / 有關聯 → H1
是否不同 → 雙尾檢定
是否提升 / 大於 → 右尾檢定
是否降低 / 小於 → 左尾檢定
看到結果後才改尾巴 → 錯誤作法
```

### Quick Check

某公司想知道新版訓練是否「提升」平均分數，`H1` 應該偏向哪種形式？

答案：右尾檢定，`H1: μ > μ0`。因為題目指定方向是「提升」。

## 3. α、β、Type I / Type II Error 與 Power 🔥🔥🔥

### 先懂一句話

檢定一定可能犯錯。`α` 是把真的 `H0` 錯誤拒絕，`β` 是把假的 `H0` 錯誤留下。

### Everyday Analogy

像公司判斷某方案是否有效。`α` 是其實沒效卻誤以為有效；`β` 是其實有效卻沒看出來。

### 在整體流程中的位置

```text
設定 α
→ 接受一定的 Type I error 風險
→ 做檢定決策
→ 同時理解 β 與 power 的取捨
```

### Key Concepts

| 真實情況 / 你的決策 | 拒絕 `H0` | 無法拒絕 `H0` |
|---|---|---|
| `H0` 真的為真 | Type I Error，`α` | 正確 |
| `H0` 其實為假 | 正確 | Type II Error，`β` |

| 名稱 | 白話 | 口訣 | 常見英文 |
|---|---|---|---|
| Type I Error, `α` | `H0` 真卻誤拒 | 誤拒真 | false positive |
| Type II Error, `β` | `H0` 假卻沒拒絕 | 誤留假 | false negative |
| Power, `1-β` | 真的有差時，成功抓到差異 | 抓真差 | statistical power |

`顯著水準（Significance Level, α）` 是事前設定的門檻，常見是 `0.05`。`α` 設得越小，越不容易亂拒絕 `H0`，但也可能更難抓到真的差異，讓 `β` 上升、`power` 下降。

### Exam Rule

```text
α → Type I Error → H0 真卻誤拒 → 誤拒真
β → Type II Error → H0 假卻沒拒絕 → 誤留假
Power → 1 - β → 真的有差時抓到差異的能力
α 設很小 → 較不易誤拒 H0，但可能降低檢定力
```

### Quick Check

`H0` 其實是假的，但檢定結果沒有拒絕 `H0`，這是哪一種錯誤？

答案：Type II Error，`β`。因為這是「誤留假」。

## 4. p-value 與結論用語 🔥🔥🔥

### 先懂一句話

`p-value` 是在 `H0` 成立的前提下，看到目前樣本結果或更極端結果的機率。`p-value` 越小，代表資料越不像 `H0` 會產生的結果。

### Everyday Analogy

像某貼文平常只有 100 次互動，今天突然 10 萬次。你會懷疑「這不像平常自然波動」。`p-value` 小，就是在說資料在 `H0` 下很不尋常。

### 在整體流程中的位置

```text
算出 t / χ² / F 統計量
→ 轉成 p-value
→ 跟 α 比較
→ 拒絕 H0 或無法拒絕 H0
```

### Key Concepts

決策規則：

```text
p-value ≤ α → 拒絕 H0
p-value > α → 無法拒絕 H0
```

例子：

```text
α = 0.05
p-value = 0.028

0.028 ≤ 0.05
→ 拒絕 H0
→ 資料提供足夠證據，支持存在統計上顯著差異
```

正確結論句型：

- 可以說：在 `α = 0.05` 下，`p-value = 0.028`，小於 `0.05`，因此拒絕 `H0`。
- 可以說：資料顯示新版前後平均分數存在統計上顯著差異。
- 不要說：`H0` 被證明是假的。
- 不要說：`p-value = 0.028` 代表 `H0` 只有 2.8% 機率為真。

| 易混淆點 | 正確理解 |
|---|---|
| `p-value` 是 `H0` 為真的機率 | 錯。它是在 `H0` 成立下，看到目前或更極端資料的機率 |
| `p-value` 很小代表效果很大 | 錯。它代表統計證據強，不直接代表效果大小 |
| `p-value > α` 代表證明 `H0` 為真 | 錯。只能說無法拒絕 `H0` |

### Exam Rule

```text
p 小於或等於 α → reject H0
p 大於 α → fail to reject H0
題目出現「證明 H0 為真」→ 通常是陷阱
題目問 p-value 意義 → 一定要加上「在 H0 成立下」
```

### Quick Check

`p-value = 0.03` 是否表示 `H0` 為真的機率是 3%？

答案：不是。`p-value` 是在 `H0` 成立下，觀察到目前或更極端結果的機率。

## 5. 信賴區間 CI 與檢定關係 🔥🔥

### 先懂一句話

`信賴區間（Confidence Interval, CI）` 是用樣本資料估計母體參數可能落在哪個範圍。考試常考它是否包含 `H0` 的假設值。

### Everyday Analogy

像估算某職缺合理薪資區間。如果整個區間都高於公司宣稱的薪資基準，你會懷疑原本基準不合理。

### 在整體流程中的位置

```text
樣本資料 → 估計參數範圍 CI → 看是否包含 H0 假設值 → 對照 p-value 結論
```

### Key Concepts

以雙尾檢定、`α = 0.05` 為例：

```text
95% CI 不包含假設值 → 通常對應 p < 0.05 → 拒絕 H0
95% CI 包含假設值 → 通常對應 p > 0.05 → 無法拒絕 H0
```

> **注意**：此 CI ↔ 拒絕 H0 的對應關係**僅適用於雙尾檢定 (two-tailed test)**。單尾檢定需用對應的單尾 CI；直接用雙尾 95% CI 判斷單尾檢定結論會得到錯誤答案。

例子：

```text
H0: μ = 70

Case A:
95% CI = [72.1, 79.4]
70 不在區間內
→ 通常拒絕 H0

Case B:
95% CI = [67.8, 73.5]
70 在區間內
→ 通常無法拒絕 H0
```

| 判斷重點 | 正確想法 |
|---|---|
| CI 是否含假設值 | 考試最常用的快判 |
| CI 寬或窄 | 可反映估計精確度，但不是本課主要決策點 |
| 95% CI | 常與 `α = 0.05` 的雙尾檢定對照 |

### Exam Rule

```text
95% CI 不含 H0 假設值 → 通常拒絕 H0
95% CI 含 H0 假設值 → 通常無法拒絕 H0
CI 是合理範圍，不是保證範圍
判斷時看「是否包含假設值」，不是只看區間寬窄
```

### Quick Check

`H0: μ = 100`，`95% CI = [102, 108]`，通常會拒絕還是無法拒絕 `H0`？

答案：通常拒絕 `H0`。因為假設值 `100` 不在信賴區間內。

## 6. t-test：平均數比較 🔥🔥🔥

### 先懂一句話

`t 檢定（t-test）` 用來處理平均數比較，尤其常見於一組平均數、兩組平均數，或同一批人的前後差異。

### Everyday Analogy

像比較考試平均分數。你可能是在問「這班平均是否等於 70 分」、「A 班和 B 班平均是否不同」，或「同一班上課前後是否進步」。

### 在整體流程中的位置

```text
題目問平均數
→ 判斷是一組、兩組獨立、還是同一批前後
→ 選 one-sample / independent two-sample / paired t-test
→ 看 p-value 做結論
```

### Key Concepts

**檢定前提假設（考試陷阱）**

| 檢定 | 前提假設 |
|---|---|
| 獨立樣本 t 檢定 | 兩組獨立、各組近似常態（或 n ≥ 30）、等變異數（Levene 檢定；若不等則改用 Welch's `equal_var=False`） |
| 配對 t 檢定 | 差值近似常態分佈 |
| 單因子 ANOVA | 各組獨立、各組近似常態、各組變異數相等（homogeneity of variance） |

情境題問「哪個檢定適合？」時，**先確認假設是否成立**，再選檢定。

| t-test 類型 | 什麼時候用 | 典型例子 | 常見 SciPy |
|---|---|---|---|
| One-sample t-test | 一組平均數 vs 某標準值 | 平均上手天數是否等於 3 天 | 常見為 `ttest_1samp` |
| Independent two-sample t-test | 兩組不同群體平均數 | A/B 廣告平均點擊數是否不同 | `ttest_ind` |
| Paired t-test | 同一批對象前後或成對資料 | 同一批員工導入工具前後處理時間 | `ttest_rel` |

配對樣本的核心不是「兩欄資料」，而是「每一筆前後可以一一對應」：

```text
員工 A：導入前 18 分 → 導入後 14 分 → 差值 -4
員工 B：導入前 22 分 → 導入後 20 分 → 差值 -2
員工 C：導入前 16 分 → 導入後 12 分 → 差值 -4

paired t-test 檢查的是：平均差值是否為 0
```

讀碼提醒：

```python
from scipy.stats import ttest_ind, ttest_rel

stat, p = ttest_ind(group_a, group_b, equal_var=True)
stat, p = ttest_rel(before, after)
```

`ttest_ind` 的 `ind` 是 independent。若 `equal_var=False`，常稱為 Welch's t-test，表示不假設兩組變異數相等。

### Exam Rule

```text
一組平均數 vs 標準值 → one-sample t-test
兩組不同人 / 不同群體平均數 → independent two-sample t-test / ttest_ind
同一批人前後 / paired data → paired t-test / ttest_rel
前後測、同用戶改版前後 → 不要選 ttest_ind
```

### Quick Check

同一批使用者在 App 改版前後各量一次操作時間，應優先想到哪種 t-test？

答案：`paired t-test`。因為資料是一一對應的同一批使用者前後比較。

## 7. χ² test 與 F-test / One-way ANOVA 🔥🔥🔥

### 先懂一句話

`χ² test` 看類別變數之間是否獨立；`F-test / one-way ANOVA` 常用來看三組以上平均數是否整體相同。

### Everyday Analogy

像選工具。你要判斷「會員等級和回購有沒有關」時用分類表；你要比較北、中、南三區平均客服分數時用三組平均數工具。

### 在整體流程中的位置

```text
題目不是 t-test
→ 若是類別 x 類別 / 是否獨立 → χ² test
→ 若是三組以上平均數 → F-test / one-way ANOVA
→ 看 p-value 做結論
```

### Key Concepts

#### χ² test of independence

`卡方檢定（Chi-square Test, χ² test）` 在本課重點是 `獨立性檢定（Test of Independence）`，用來判斷兩個類別變數是否有關聯。

例子：

```text
                有回購   無回購
白金會員          80       20
一般會員          45       55

H0：會員等級與是否回購獨立
H1：會員等級與是否回購不獨立
```

卡方檢定比較的是：

```text
Observed Count 實際觀察次數
vs
Expected Count 若真的獨立時理論上應該出現的次數
```

若實際次數和期望次數差很多，`χ²` 變大，`p-value` 變小，越可能拒絕 `H0`。

讀碼提醒：

```python
from scipy.stats import chi2_contingency

stat, p, dof, expected = chi2_contingency(observed_table)
```

`observed_table` 應是計數表，不是直接丟百分比。

> **重要限制**：當列聯表中任一格的期望次數 (expected count) < 5 時，卡方檢定結果不可靠，應改用 **Fisher's Exact Test**。

#### F-test / One-way ANOVA

`F 檢定（F-test）` 在本課最重要的應用是 `單因子變異數分析（One-way ANOVA）`，用來比較三組以上平均數是否整體相同。

```text
H0: μ1 = μ2 = μ3 = ...
H1: 至少有一組平均數不同
```

`F` 的直觀：

```text
F = MS_between / MS_within（組間均方 / 組內均方）
```

| 情況 | 直觀結果 |
|---|---|
| 組平均很接近，組內波動又大 | `F` 小，`p` 大 |
| 組平均差很多，組內又穩定 | `F` 大，`p` 小 |

重要限制：如果 ANOVA 拒絕 `H0`，只能說「至少有一組平均數不同」，不能直接說是哪兩組不同。需使用**事後比較 (Post-hoc Test)**，例如 **Tukey HSD**，找出差異所在的組別對。考試題目需能識別此名詞，計算不在本課範圍內。

讀碼提醒：

```python
from scipy.stats import f_oneway

stat, p = f_oneway(group_a, group_b, group_c)
```

### Exam Rule

```text
類別資料 / 列聯表 / 是否獨立 → χ² test of independence
χ² test 的輸入 → observed counts，不是百分比
三組以上平均數 → F-test / one-way ANOVA
ANOVA 拒絕 H0 → 至少一組不同，不代表知道哪一組不同
```

### Quick Check

題目問「會員等級」與「是否回購」是否有關，資料整理成列聯表，應選哪個檢定？

答案：`χ² test of independence`。因為兩個變數都是類別型，且題目問是否獨立 / 有關聯。

## 8. Exam Decision Trees 🔥🔥🔥

### 8.1 先決定要不要拒絕 H0

```text
題目給 p-value 與 α？
│
├─ p-value ≤ α
│  └─ 拒絕 H0
│
└─ p-value > α
   └─ 無法拒絕 H0
```

### 8.2 用 CI 快判

```text
題目給 95% CI 與 H0 假設值？
│
├─ 假設值在 CI 裡
│  └─ 通常無法拒絕 H0
│
└─ 假設值不在 CI 裡
   └─ 通常拒絕 H0
```

### 8.3 選檢定方法

```text
先問：題目要比較什麼？
│
├─ 平均數？
│  │
│  ├─ 一組平均數 vs 標準值
│  │  └─ one-sample t-test
│  │
│  ├─ 兩組平均數
│  │  │
│  │  ├─ 不同人 / 不同群體
│  │  │  └─ independent two-sample t-test
│  │  │
│  │  └─ 同一批人前後 / 成對資料
│  │     └─ paired t-test
│  │
│  └─ 三組以上平均數
│     └─ F-test / one-way ANOVA
│
└─ 類別 x 類別 / 是否獨立？
   └─ χ² test of independence
```

### 8.4 看題目用語選尾巴

```text
題目描述 H1 方向？
│
├─ 是否不同 / 是否改變
│  └─ 雙尾檢定
│
├─ 是否提升 / 是否大於
│  └─ 右尾檢定
│
└─ 是否降低 / 是否小於
   └─ 左尾檢定
```

## 9. Trap Clinic 🔥🔥🔥

### Trap 1：p-value 是 H0 為真的機率

錯。`p-value` 是在 `H0` 成立下，看到目前或更極端結果的機率。

Exam fix：

```text
題目問 p-value 意義 → 一定回答「在 H0 成立下」
```

### Trap 2：p > α 就證明 H0 為真

錯。`p > α` 只能說「無法拒絕 `H0`」，不是證明 `H0` 正確。

Exam fix：

```text
p > α → fail to reject H0
看到「證明 H0 為真」→ 多半是錯選項
```

### Trap 3：拒絕 H0 就代表 H1 被完全證明

錯。拒絕 `H0` 代表資料提供足夠證據支持 `H1`，但不是數學上的完全證明。

Exam fix：

```text
reject H0 → 有統計證據支持 H1
完全確定 / 證明 / 必然 → 注意陷阱
```

### Trap 4：α 和 β 方向背反

錯。`α` 是 `H0` 真卻誤拒；`β` 是 `H0` 假卻沒拒絕。

Exam fix：

```text
α → 誤拒真
β → 誤留假
power → 1 - β
```

### Trap 5：前後測資料用 ttest_ind

錯。`ttest_ind` 是兩組獨立樣本。前後測、同一批人、同用戶改版前後，應先想到 paired t-test。

Exam fix：

```text
same users before/after → paired t-test / ttest_rel
different groups → independent t-test / ttest_ind
```

### Trap 6：χ² test 可以直接吃百分比

錯。卡方獨立性檢定通常需要觀察到的計數表（observed counts），不是直接把百分比丟進去。

Exam fix：

```text
列聯表 + count → χ² test
只有百分比 → 小心，需回到計數資料
```

### Trap 7：ANOVA 顯著就知道哪一組不同

錯。One-way ANOVA 顯著只能說「至少有一組平均數不同」，不能直接判斷是哪兩組不同。

Exam fix：

```text
F-test / ANOVA reject H0 → 至少一組不同
哪組不同 → 需要後續比較，本課不展開
```

### Trap 8：看到結果再決定單尾或雙尾

錯。單尾或雙尾應根據研究問題事前決定，不能為了顯著才改方向。

Exam fix：

```text
是否不同 → 雙尾
是否提升 / 降低 → 單尾
看完結果才改尾巴 → 錯
```

## 10. Practice Questions

### 10.1 H0 / H1 與結論用語

**Q1.** 題目問新版流程是否「改變」平均送達時間，沒有指定變快或變慢，通常是單尾還是雙尾？

答案：雙尾。

理由：題目只問「是否改變 / 是否不同」，不指定方向。

**Q2.** `p-value = 0.04`、`α = 0.05`，應該如何決策？

答案：拒絕 `H0`。

理由：`0.04 ≤ 0.05`。

**Q3.** `p-value = 0.12`、`α = 0.05`，可以說「證明 H0 為真」嗎？

答案：不可以。

理由：只能說「無法拒絕 `H0`」，不是證明 `H0` 為真。

**Q4.** `p-value = 0.02` 正確代表什麼？

答案：在 `H0` 成立下，觀察到目前或更極端結果的機率是 0.02。

理由：`p-value` 不是 `H0` 為真的機率。

### 10.2 α、β、Power 與 CI

**Q5.** `H0` 其實為真，但研究者拒絕了 `H0`，這是什麼錯誤？

答案：Type I Error，`α`。

理由：`α = 誤拒真`。

**Q6.** `H0` 其實為假，但檢定沒有拒絕 `H0`，這是什麼錯誤？

答案：Type II Error，`β`。

理由：`β = 誤留假`。

**Q7.** `Power` 等於什麼？

答案：`1 - β`。

理由：Power 是真的有差異時，成功偵測到差異的能力。

**Q8.** `H0: μ = 50`，`95% CI = [52, 58]`，通常對應拒絕或無法拒絕 `H0`？

答案：拒絕 `H0`。

理由：假設值 `50` 不在區間內。

**Q9.** `H0: μ = 50`，`95% CI = [48, 55]`，通常對應拒絕或無法拒絕 `H0`？

答案：無法拒絕 `H0`。

理由：假設值 `50` 在區間內。

### 10.3 檢定方法選擇

**Q10.** 某平台宣稱平均等待時間是 5 分鐘，抽一批訂單檢查平均等待時間是否等於 5 分鐘，應選什麼？

答案：One-sample t-test。

理由：一組樣本平均數與一個標準值比較。

**Q11.** 比較 A 廣告組與 B 廣告組的平均轉單金額，兩組使用者互不重疊，應選什麼？

答案：Independent two-sample t-test。

理由：兩組獨立群體的平均數比較。

**Q12.** 同一批員工導入新系統前後各測一次平均處理時間，應選什麼？

答案：Paired t-test。

理由：同一批人前後資料是一一配對。

**Q13.** 比較北、中、南三區客服平均滿意度是否相同，應選什麼？

答案：F-test / One-way ANOVA。

理由：三組以上平均數的整體比較。

**Q14.** 檢查會員等級與是否回購是否獨立，應選什麼？

答案：Chi-square test of independence。

理由：兩個變數都是類別型，且題目問是否獨立 / 有關聯。

**Q15.** `chi2_contingency` 的輸入應該是什麼？

答案：觀察到的計數列聯表。

理由：卡方檢定比較 observed counts 與 expected counts，不是直接使用百分比。

### 10.4 Mixed Traps

**Q16.** One-way ANOVA 的 `p-value ≤ α`，可以直接說「第 1 組比第 3 組高」嗎？

答案：不可以。

理由：ANOVA 只能先說至少有一組平均數不同，不指出哪一組不同。

**Q17.** 題目出現 `scipy.stats.ttest_ind`，最可能對應哪種情境？

答案：兩組獨立樣本平均數比較。

理由：`ind` 代表 independent。

**Q18.** 題目出現 `scipy.stats.ttest_rel(before, after)`，最可能對應哪種情境？

答案：配對樣本 t-test。

理由：`rel` 表示 related，常見於同一批對象前後比較。

**Q19.** 題目只說「新方法是否讓平均分數不同」，可以因為樣本平均較高就改成右尾檢定嗎？

答案：不可以。

理由：單尾或雙尾要依研究問題事前決定，不能看結果後調整。

**Q20.** `α = 0.01` 比 `α = 0.05` 更嚴格，這一定比較好嗎？

答案：不一定。

理由：較小的 `α` 會降低誤拒 `H0` 的機率，但也可能讓真的差異更難被偵測到。

## Final Oral Recall

考前最後 3 分鐘，把這幾句唸一次：

1. 假設檢定先從 `H0` 開始，`H0` 通常代表無差異、無效果、無關聯。
2. `p-value ≤ α` 就拒絕 `H0`，`p-value > α` 就無法拒絕 `H0`。
3. `p-value` 不是 `H0` 為真的機率，而是在 `H0` 成立下看到目前或更極端結果的機率。
4. `α = 誤拒真`，`β = 誤留假`，`power = 1 - β`。
5. `95% CI` 不含假設值通常拒絕 `H0`，包含假設值通常無法拒絕 `H0`。
6. 平均想 `t`，類別想 `χ²`，三組以上平均想 `F / ANOVA`。
7. 前後測、同一批人、成對資料要想到 `paired t-test`，不要誤選 `ttest_ind`。
