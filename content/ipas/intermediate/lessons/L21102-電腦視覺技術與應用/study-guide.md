## Section 3. Core Concepts (核心觀念)

### 3.1 為何需要 CNN？從全連接到卷積的進化

先別急著學卷積，先搞懂**為什麼影像不能直接餵給全連接網路（Fully Connected Network, FCN）**。

**問題一：參數爆炸。** 一張 `224×224×3` 的彩色影像攤平（flatten）後是 `150,528` 個輸入。如果第一層就接 1,000 個神經元，光這一層就要 **1.5 億**個權重。你家的 GPU（或 iPAS 考場想像中的筆電）會先爆。

🗣️ 白話說明：這就像你要幫台北大巨蛋 10 場滿場演唱會的觀眾（約 15 萬人）每人都建一個好友關係欄位，還沒傳訊息資料庫就先炸了。

**問題二：對位置敏感。** FC 把像素當獨立欄位，貓移到右邊三個像素，整組權重要重學。但貓還是貓啊。

**卷積神經網路（Convolutional Neural Network, CNN）用兩個關鍵假設解決這件事：**

1. **局部性 (Locality)**：影像的關鍵訊號在**一小塊區域**內（邊緣、紋理），不需要整張圖的每個像素都互動。
2. **平移不變性 (Translation Invariance)**：同一個特徵偵測器（例如「直邊偵測器」）應該能在影像**任何位置**都有效。

🗣️ 白話說明：想像你在 IG 限時動態滑過一堆貓照片。你的眼睛不是每張圖都從左上掃到右下逐格比對，而是「貓耳朵形狀到處都算貓耳朵」。CNN 做的就是同一件事——用同一個小型偵測器（kernel）掃過全圖。

### 3.2 CNN 四大建構元件

> 📊 **視覺化：** CNN 完整前向傳播流程圖 → 詳見 [diagrams/02-cnn-forward-pass.md](diagrams/02-cnn-forward-pass.md)

一個標準 CNN = 「卷積 → 激活 → 池化」堆疊 N 次 + 最後接全連接層。

```
輸入影像                   特徵圖                    分類結果
[224×224×3] ──卷積──► [特徵圖] ──池化──► [縮小版特徵圖] ──...──► [全連接] ──► [貓 / 狗 / 其他]
                 ▲                   ▲                      ▲
              抽特徵             降維度                    做決策
```

#### 3.2.1 卷積層 (Convolutional Layer)

**卷積核（kernel / filter）**：一個小矩陣（通常 3×3 或 5×5），像一個「特徵偵測印章」，在影像上滑動、逐塊做內積。

```
輸入 5×5                   3×3 kernel               輸出 3×3
┌─┬─┬─┬─┬─┐                ┌─┬─┬─┐                ┌─┬─┬─┐
│ │ │ │ │ │                │ │ │ │                │ │ │ │
├─┼─┼─┼─┼─┤                ├─┼─┼─┤     滑動、內積   ├─┼─┼─┤
│ │ │ │ │ │   ⊛            │ │ │ │   ══════════►  │ │ │ │
├─┼─┼─┼─┼─┤                ├─┼─┼─┤                ├─┼─┼─┤
│ │ │ │ │ │                │ │ │ │                │ │ │ │
├─┼─┼─┼─┼─┤                └─┴─┴─┘                └─┴─┴─┘
│ │ │ │ │ │
├─┼─┼─┼─┼─┤
│ │ │ │ │ │
└─┴─┴─┴─┴─┘
```

**四個你必記的參數：**

| 參數 | 中文 | 意義 |
|---|---|---|
| `F` | kernel 大小 | filter 的邊長（3×3 就填 3） |
| `S` | stride 步長 | 每次滑幾格；S=2 會讓輸出縮小一半 |
| `P` | padding 填補 | 在邊緣補幾圈 0；"valid"=0，"same"=保持尺寸 |
| — | 感受野 (receptive field) | 輸出某格「看得到」的輸入範圍；越深越大 |

**🔥🔥 卷積輸出尺寸公式（必背，出題最高頻之一）：**

```
O = ⌊(W − F + 2P) / S⌋ + 1
```

- `W`：輸入邊長
- `F`：kernel 大小
- `P`：padding
- `S`：stride
- `⌊ ⌋`：**floor（向下取整）**——PyTorch 與 TensorFlow 的預設行為
- 一般設計會讓 `(W − F + 2P)` 剛好被 `S` 整除；若不整除，框架以 floor 處理。

**📏 實戰算例（直接拿 ResNet-50 的第一層當題目）：**

輸入 224×224×3 影像，用 7×7 卷積核、stride 2、padding 3、64 個 filter，請問輸出尺寸？

```
O = ⌊(224 − 7 + 2×3) / 2⌋ + 1
  = ⌊(224 − 7 + 6) / 2⌋ + 1
  = ⌊223 / 2⌋ + 1
  = ⌊111.5⌋ + 1
  = 111 + 1
  = 112 ✓
```

> 📌 **小提醒：** 當 `W − F + 2P` 不能被 `S` 整除時，PyTorch/TensorFlow 採用 floor（向下取整）—— ResNet-50 官方 conv1 就是這種情況（223 / 2 = 111.5 → floor 取 111）。

輸出：**112 × 112 × 64**（這就是 ResNet-50 `conv1` 的實際尺寸）。

**👉 Same padding 快捷規則（S=1 時）：** `P = (F − 1) / 2`
- F=3 → P=1
- F=5 → P=2
- F=7 → P=3

> 💡 **延伸：空洞卷積 (Dilated / Atrous Convolution)** — 在 kernel 元素之間加入「間隔」（dilation rate），讓感受野放大而不增加參數量。常用於語意分割網路（DeepLab）。iPAS 目前題庫較少直接測，但看到「擴大感受野又不想多 pooling」的描述要能認出。

**📐 參數量計算（中級偶爾會考）：**

```
一個 conv 層的參數 = (F × F × C_in + 1) × C_out
                                      ▲
                                    bias
```

範例：3×3 卷積、輸入 64 channel、輸出 128 channel → `(3×3×64 + 1) × 128 = 73,856` 個參數。

🗣️ 白話說明：kernel 就像外送員手上的「檢查表」，每次掃到一個路口（感受野）就打一個分數。stride 是「每隔幾條巷子才停一次」，padding 是「路口不夠就在外圍畫假路」。

#### 3.2.2 池化層 (Pooling Layer) 🔥

池化做的事很單純：**把特徵圖縮小**。但它的目的常常被考生記錯，請跟我唸三遍：

> **池化層不是為了抽特徵，是為了降維度、保留重要訊號、提升平移不變性。**

**最大池化 (Max Pooling) — 最常用：**

```
輸入 4×4                    2×2 max pool, stride=2       輸出 2×2
┌──┬──┬──┬──┐                                         ┌────┬────┐
│ 1│ 3│ 2│ 4│                                          │ 3  │ 4  │
├──┼──┼──┼──┤                ─────────────────►        ├────┼────┤
│ 5│ 6│ 1│ 8│                                          │ 6  │ 9  │
├──┼──┼──┼──┤                每 2×2 視窗取最大值          └────┴────┘
│ 2│ 4│ 7│ 9│
├──┼──┼──┼──┤
│ 1│ 3│ 5│ 2│
└──┴──┴──┴──┘
```

**平均池化 (Average Pooling)**：同樣視窗，取平均值（較平滑、較少銳利邊緣）。

**全域平均池化 (Global Average Pooling, GAP)**：把整張 `H×W` 特徵圖壓成一個數。ResNet 在最後一層 conv 後、FC 前就是用這個取代 flatten。

🗣️ 白話說明：池化像你整理 Google 相簿時每 4 張只留 1 張代表作。你沒學到新東西，但相簿變小、重要畫面還在，而且「貓移到左邊 3 像素」這種小平移不會讓代表作改變——這就是**平移不變性**。

---

#### 3.2.3 激活函數 (Activation Function)

CNN 裡最常用 **ReLU (Rectified Linear Unit)**：`ReLU(x) = max(0, x)`。

```
  輸出
   ▲
   │       ╱
   │      ╱
   │     ╱
───┼────────► 輸入
   0
```

目的：引入**非線性（non-linearity）**。沒有激活函數，再多層卷積堆疊也等於一層線性轉換，白堆。

---

#### 3.2.4 全連接層 (Fully Connected Layer)

把最後的特徵圖 **flatten（攤平）**成一維向量，接全連接分類器，輸出 N 個類別機率（通常接 softmax）。

```
[7×7×512 特徵圖] ── flatten ──► [25088 維向量] ── FC ──► [1000 類別機率]
```

🗣️ 白話說明：前面都在「長眼睛」（抽特徵），最後這一層才是「動嘴巴」（說答案）。

---

### 3.3 ResNet 與殘差連接 🔥

> 📊 **視覺化：** ResNet 殘差塊結構圖 → 詳見 [diagrams/03-resnet-skip-connection.md](diagrams/03-resnet-skip-connection.md)

**問題：退化 (degradation problem)。** He et al. 2015 論文發現，把網路從 20 層堆到 56 層，**連訓練誤差都變大**。這不是過擬合（overfitting），是優化不起來——梯度在深層傳不到、網路學不到「恆等映射」。

> 🧠 **為什麼「學恆等映射」這麼難？** 重複疊加的非線性（例如 ReLU）本身會扭曲訊號——要讓很深的一整疊 layer 剛好把輸入原封不動傳過去，對 SGD（隨機梯度下降）來說是非常難的優化目標。加上 skip connection (`H(x) = F(x) + x`) 等於直接告訴網路：「你只要學殘差 F(x) 就好，恆等部分我幫你走捷徑」，把難學的目標重新定義成容易學的目標。

**解法：殘差連接 (residual / skip connection)。**

```
   ┌────────── 恆等 (identity shortcut) ──────────┐
   │                                              │
   │                                              ▼
輸入 x ──► [Conv → BN → ReLU → Conv → BN] ──► (+) ──► ReLU ──► 輸出 H(x)
                        F(x)
```

**公式**：`H(x) = F(x) + x`

- `F(x)` 是「這個 block 學到的殘差」
- `+ x` 是恆等捷徑
- 當這一層不需要學新東西時，只要 `F(x) → 0`，整個 block 就變成恆等映射——網路最差就是「這一層沒做事」，不會倒退。

**白話收益：**
1. 梯度有「高速公路」可以直接反傳（解決梯度消失 vanishing gradient）
2. 讓「加深不會變差」變成網路的 default 行為
3. ResNet-152 在 ImageNet 拿到 3.57% top-5 error，直接贏了 2015 年 ILSVRC（此為 ResNet ensemble 在 ILSVRC 2015 的提交結果；單一 ResNet-152 模型大約為 4.49%）

🗣️ 白話說明：想像你用 LINE 傳訊息給阿嬤，中間經過 10 個親戚轉達。殘差連接就是多拉一條「直接把原話一起附上」的捷徑，就算中間親戚轉譯出錯，原話還在。

─────────────────────────              ─────────────────────────

|---|---|---|---|---|

🗣️ 白話說明（Uber Eats 類比）：
- 分類 = 「這張外送照片裡有食物嗎？」
- 偵測 = 「用方框圈出便當、珍奶、餐具各在哪」
- 語意分割 = 「這幾個像素是便當、那幾個像素是珍奶」（同類不分杯）
- 實例分割 = 「三杯珍奶要分成珍奶 1、珍奶 2、珍奶 3」

**🎯 關鍵記憶法「一張標 → 有框 → 每個像素」**：從分類（一張標一個類）、到偵測（有框有類），再到分割（每個像素都有類）——難度和資訊量逐階遞增。

### 3.5 YOLO 深入解析 🔥

**YOLO = You Only Look Once**。單階段（single-shot）偵測器的代表。

**核心想法：** 把整張圖切成 `S×S` 的格子（grid），每格直接預測 **(bbox 座標, 類別, 信心)**。一次 forward pass 結束，速度極快。

```
原圖                     切成 grid                預測輸出
┌─────────────┐         ┌─┬─┬─┬─┬─┬─┬─┐       每格：
│             │         ├─┼─┼─┼─┼─┼─┼─┤       - bbox (x,y,w,h)
│   🐕    🐑   │   ──►   ├─┼─🐕┼─┼─🐑┼─┤  ──►  - 類別機率
│       🐑 🐑 │         ├─┼─┼─┼─┼─┼─┼─┤       - 信心分數
│             │         ├─┼─┼─┼─🐑🐑─┼─┤
└─────────────┘         └─┴─┴─┴─┴─┴─┴─┘
```

**YOLO 世代演進（考試口訣：anchor-based → anchor-free → NMS-free）：**

| 版本 | 關鍵特色 |
|---|---|
| v3–v7 | **Anchor-based**（預設 prior box，要調）、有 objectness 分支 |
| **v8** | **Anchor-free**（直接預測中心點到四邊距離）、CSPDarknet + C2f 模組 backbone（以 C2f 取代 YOLOv5 的 C3）、PANet neck、損失 = BCE + CIoU + DFL、仍需 **NMS** 後處理 |
| **v10** | **NMS-free**（Consistent Dual Assignments：訓練時一對多、推論時一對一，不用 NMS 後處理）|

**⚠️ iPAS 重點：** 題目通常只用「YOLO」通稱，不會考你 v8 還 v10 差在哪。但「單階段、快、即時」是必背標籤。

**NMS (Non-Maximum Suppression, 非最大值抑制)**：偵測器會對同一物件產生多個重疊 bbox，NMS 用 IoU 門檻（常 0.45–0.7，依版本／任務而定；Ultralytics YOLOv8 預設 IoU=0.7、conf=0.25）把重疊且信心較低的 bbox 刪掉。YOLOv10 用 one-to-one head 免掉這步。

🗣️ 白話說明：YOLO 就像你打開 Google 地圖找餐廳，整張地圖一瞥就看到哪裡有便當店、哪裡有手搖杯。Faster R-CNN 則像先掃一遍地圖畫出所有候選地點，再對每個地點逐一細查——慢，但精準。

> 💡 **台灣製造業觀察：** YOLOv4（由台灣中研院廖弘源教授團隊所開發）與 YOLOv7 在台灣 AOI 自動光學檢測、工廠瑕疵偵測場景中極為普及。iPAS 情境題若出現「產線即時檢測」「工廠缺陷辨識」一類的本土情境，YOLO 家族幾乎是標準答案。

### 3.7 評估指標 🔥🔥

> 📊 **視覺化：** IoU 視覺化 + mAP@0.5 vs mAP@0.5:0.95 對比 → 詳見 [diagrams/05-iou-map-metrics.md](diagrams/05-iou-map-metrics.md)

#### 3.7.1 IoU — 交集比聯集 (Intersection over Union)

**公式：**

```
IoU = Area(A ∩ B) / Area(A ∪ B)
```

範圍 `[0, 1]`，通常 **IoU ≥ 0.5** 視為一個正確偵測 (True Positive)。

**📏 算例（ASCII 實戰）：**

預測 bbox = 100×100（左上角在原點 (0,0)）
真實 bbox = 100×100（左上角在 (50, 50)）

```
     0    50   100  150
   0 ┌─────┬────┐
     │  A  │    │                A = 預測 bbox
  50 │  ┌──┼────┼──┐             B = ground truth bbox
     │  │ ∩│    │  │             ∩ = 交集（50×50 小方格）
 100 └──┼──┘  B │  │
        │        │  │
 150    └────────┘
```

**計算：**
- 交集：50 × 50 = **2,500**
- A 面積 = 100×100 = 10,000；B 面積 = 10,000
- 聯集：10,000 + 10,000 − 2,500 = **17,500**
- IoU = 2,500 / 17,500 ≈ **0.143**

結論：在 0.5 門檻下，這個預測是 **False Positive**（沒過門檻）。

🗣️ 白話說明：IoU 就是你跟朋友一起租房，兩人房間的「共用區」除以「合起來的總區域」。共用越多比例越高，分數越高。

---

#### 3.7.2 Precision / Recall / F1

```
Precision = TP / (TP + FP)    ← 你說有的東西，有多少是真的
Recall    = TP / (TP + FN)    ← 真的有的東西，你抓到幾個
F1        = 2PR / (P + R)     ← 精確率與召回率的調和平均
```

---

#### 3.7.3 AP 與 mAP

- **AP (Average Precision)**：針對**某一個類別**，畫出 Precision–Recall 曲線，取**曲線下面積**（COCO 用 101 點插值）。
- **mAP (mean AP)**：**各類別 AP 的平均**。物件偵測的標準頭牌指標。

---

#### 3.7.4 mAP@0.5 vs mAP@0.5:0.95 🔥🔥

| 指標 | 定義 | 嚴格度 | 來源 |
|---|---|---|---|
| **mAP@0.5** | 單一 IoU 門檻 = 0.5 時的 mAP | 寬鬆 | PASCAL VOC |
| **mAP@0.5:0.95** | IoU 門檻從 0.5 到 0.95，每 0.05 一階共 **10 個門檻**，各算一次 mAP 後**平均** | 嚴格 | **COCO**（現代論文標配） |

```
mAP@0.5         = mAP(IoU=0.5)

mAP@0.5:0.95    = mean{ mAP(0.50), mAP(0.55), mAP(0.60), …, mAP(0.95) }
                         ▲                                ▲
                       10 個門檻（step 0.05）            全部平均
```

🗣️ 白話說明：mAP@0.5 像「只要寫得跟標準答案差不多就給分」，mAP@0.5:0.95 像「從寬鬆到超嚴格 10 個老師一起打分、再平均」——哪個看起來比較公正？當然是後者。

## Section 5. 口訣 / Mnemonics

1. **CNN 三件事：「抽特徵、降維度、做決策」** — 卷積抽特徵、池化降維度、全連接做決策。
2. **任務三階梯：「一張標 → 有框 → 每個像素」** — 分類 → 偵測 → 分割，資訊量逐階遞增。
3. **池化四字訣：「縮、留、平、降」** — **縮**小尺寸、**留**住訊號、**平**移不變、**降**低參數。陷阱題選「抽特徵」就死了。
4. **卷積輸出公式：「減 F 加 2P、除 S 加 1」** — `O = ⌊(W − F + 2P) / S⌋ + 1`（框架預設 floor 向下取整）。口訣順著念就對。
5. **IoU 記憶：「交／聯要過半」** — 交集 ÷ 聯集，≥ 0.5 才算正確偵測。
6. **YOLO 口訣：「一眼看遍，快不一定輸」** — single-shot ≠ 精度低，新版 YOLO 已追平兩階段。
7. **ResNet 口訣：「加條捷徑、最差不倒退」** — skip connection 確保網路最爛也只是恆等映射。
8. **mAP 門檻記憶：「五到九五，每五一站，共十站」** — COCO mAP@0.5:0.95 共 10 個門檻。
9. **三大分割辨識：「語意不認人、實例會點名、全景全包辦」** — semantic 不分個體、instance 分個體、panoptic 連天空都標。

## Section 6. 考試陷阱 (Exam Traps)

> ❌ **陷阱 1：池化層的目的是抽取更多特徵。**
> ✅ **正解：** 池化層的目的是「**縮小特徵圖尺寸＋保留關鍵訊號＋降低參數量＋提升平移不變性**」。抽特徵是**卷積層**做的事。這是 CCChen 現場回報的重複題。

---

> ❌ **陷阱 2：YOLO 是 CNN 的替代品。**
> ✅ **正解：** YOLO 是「**建立在 CNN backbone 之上的偵測頭架構**」。YOLOv8 的 backbone 就是 CSPDarknet + C2f 模組（CNN 家族）。CNN 是底層元件、YOLO 是上層應用，兩者是**層級關係**不是替代。

---

> ❌ **陷阱 3：語意分割和實例分割是同一件事。**
> ✅ **正解：** 語意分割只標「**像素的類別**」（同類不分個體）；實例分割標「**像素的類別＋每個個體各自的 mask**」。照片裡 3 隻羊：語意分割一塊「羊」色、實例分割要分成羊₁、羊₂、羊₃。iPAS 114/05 已考過。

---

> ❌ **陷阱 4：mAP 就是單一 IoU 門檻下的平均精度。**
> ✅ **正解：** 不一定。`mAP@0.5` 是單門檻，但 COCO 的 `mAP@0.5:0.95` 是「**10 個門檻（0.5, 0.55, …, 0.95）的平均**」。後者更嚴格、更常見於現代論文。

---

> ❌ **陷阱 5：兩階段偵測器（Faster R-CNN）一定比單階段（YOLO）更準。**
> ✅ **正解：** 傳統上是，但**新版 YOLO（v8、v10）已大幅追平甚至超越**。iPAS 如果題目寫「兩階段**一定**比較精準」，要警覺這可能是陷阱——實務上速度已不是必然的精度代價。

---

> ❌ **陷阱 6：卷積層輸出尺寸 = W / S。**
> ✅ **正解：** 正確公式 `O = ⌊(W − F + 2P) / S⌋ + 1`（框架預設 floor 向下取整）。**漏掉 padding 和 kernel 減項**是最常見錯誤。

---

> ❌ **陷阱 7：ResNet 的 skip connection 是為了解決過擬合 (overfitting)。**
> ✅ **正解：** skip connection 解決的是**退化問題 (degradation) 與梯度消失 (vanishing gradient)**，不是過擬合。退化問題的特徵是「訓練誤差也變大」，和過擬合「訓練低、測試高」完全不同。

---

> ❌ **陷阱 8：IoU ≥ 0.5 就算是完美偵測。**
> ✅ **正解：** IoU ≥ 0.5 只是**被計為 True Positive 的最低門檻**，不代表「完美」。COCO 嚴格評測會要求到 0.95。更高 IoU 代表定位更精準。

---

## Section 7. 情境題快速判斷 (Scenario Quick-Judge)

考試當天翻這一頁——看到關鍵字直接勾答案：

```
🔑 關鍵字查找表

1. 「即時偵測 / 毫秒級 / 邊際裝置 / 監視器 / 30 FPS」
   → YOLO (single-shot detector)

2. 「最高精度 / 醫療影像 / 不計時間成本 / 小物件」
   → Faster R-CNN (two-stage detector)

3. 「醫學影像切邊 / 細胞邊界 / 腫瘤輪廓 / 像素級標註」
   → 語意分割 (Semantic Segmentation), U-Net

4. 「同類但要分個體 / 每人獨立 mask / 人群計數」
   → 實例分割 (Instance Segmentation), Mask R-CNN

5. 「只要分一個類別 / 貓還是狗 / 有沒有人」
   → 影像分類 (Image Classification), ResNet

6. 「有物件＋要位置＋要類別」
   → 物件偵測 (Object Detection), bbox 輸出

7. 「平移不變性 / 降低參數量 / 縮小特徵圖」
   → 池化層 (Pooling Layer)，最常見 max pooling

8. 「梯度消失 / 網路很深卻訓不起來 / 退化問題」
   → 加 skip connection (ResNet 殘差連接)

9. 「交集比聯集 / 預測框和真實框重疊程度」
   → IoU (Intersection over Union)

10. 「10 個門檻平均 / 嚴格評測 / COCO 標準」
    → mAP@0.5:0.95

11. 「同時處理「天空、路面」和「每個人、每台車」」
    → 全景分割 (Panoptic Segmentation)

12. 「重疊 bbox 太多、留一個信心最高的」
    → NMS (Non-Maximum Suppression)

13. 「anchor-free + NMS-free + 端對端部署」
    → YOLOv10

14. 「Conv → BN → ReLU → Conv → BN → (+) x」
    → ResNet 殘差 block
```
