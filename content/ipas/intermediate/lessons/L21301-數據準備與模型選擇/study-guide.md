## Section 3: Core Concepts
### 3.1 先抓主軸：資料準備比模型更早決定成敗
`數據準備（Data Preparation）` 是把原始資料整理成可學、可比、可部署的形式。
`模型選擇（Model Selection）` 是依任務、資料、限制條件挑出最合理的方法。
很多人會先問「哪個模型最強」。
中級考生要先問：
- 資料有沒有代表性？
- 欄位能不能直接用？
- 切分方法公平嗎？
- 有沒有資料洩漏（Data Leakage）？
🗣️ 白話說明：這像租屋不是先看哪間照片最漂亮，而是先看地點、預算、通勤、漏水。模型很強，但資料爛，結果一樣爛。
🔥🔥 必記：資料品質常比模型複雜度更影響成效。

### 3.2 數據收集（Data Collection）
`數據收集（Data Collection）` 是取得資料並確認它是否真的適合這個任務。
看四件事：
- 任務對不對
- 代表性夠不夠
- 標籤穩不穩
- 合法能不能用

#### 3.2.1 任務先行
若是 `分類（Classification）`，通常需要類別標籤（Label）。
若是 `回歸（Regression）`，通常需要連續數值目標。
若是 `分群（Clustering）`，可沒有標籤，但仍要有能區分樣本的特徵。
```text
業務問題
  ↓
AI 任務型態
  ↓
需要哪些欄位
  ↓
是否有標籤 / 目標值
```
🗣️ 白話說明：你想預測 104 履歷會不會過關，需要「是否錄取」這種標籤；你想預估薪資，就要有薪資數字。問題不同，收的資料不同。

#### 3.2.2 代表性（Representativeness）
`代表性（Representativeness）` 指樣本能不能反映實際上線情境。
常見問題：
- 只收單一地區
- 只收某時段
- 只收成功案例
- 缺少新客或少數族群資料
🗣️ 白話說明：只看台北市信義區外送紀錄，卻想推論全台需求，很容易偏掉。
🔥 考點：題目若提到樣本偏向特定族群、地區、時間，多半在考代表性不足。

#### 3.2.3 標籤品質（Label Quality）
`標籤品質（Label Quality）` 指目標欄位本身是否正確、一致、可信。
常見問題：
- 人工標註標準不一致
- 舊制度留下來的標籤不適合現在
- 目標欄位其實不是模型真正要預測的事
🗣️ 白話說明：像分組報告有人把「有回貼圖」算有參與，有人不算。標準一亂，標籤就不可靠。

### 3.3 數據清洗與預處理（Data Cleaning and Preprocessing）🔥🔥
`數據清洗與預處理（Data Cleaning and Preprocessing）` 是把資料中的錯誤、缺漏、不一致、尺度差異處理掉。
典型流程：
```text
原始資料
  ↓
缺失值
  ↓
異常值
  ↓
重複值 / 格式統一
  ↓
縮放 / 編碼
  ↓
可建模資料
```
🗣️ 白話說明：像整理 LINE 投票結果，要先刪重複回覆、補空白、統一日期格式，不然後面分析會越看越亂。

### 3.4 缺失值（Missing Values）
`缺失值（Missing Values）` 指欄位空白或沒有值。
常見處理：
- 刪除（Deletion）
- 補值（Imputation）
- 保留並視為一種訊號

#### 3.4.1 什麼時候刪除
適合：
- 缺失比例低
- 這些樣本刪掉後不影響代表性
- 欄位重要性不高

#### 3.4.2 什麼時候補值
適合：
- 資料量珍貴
- 缺失比例不低
- 刪除代價太高
常見補法：
- 數值欄位：平均數（Mean）或中位數（Median）
- 類別欄位：眾數（Mode）
🔥 考點：有極端值時，中位數（Median）通常比平均數（Mean）更穩。
🗣️ 白話說明：像公司午餐統計 50 人裡有 1 人沒填，可以刪；如果 20 人沒填，直接刪就不合理。

#### 3.4.3 缺失本身可能是訊號
有些空白不是單純沒填，而是有意義。
例如：
- 未填收入，可能代表不願透露
- 沒有點擊紀錄，可能代表根本沒看到推薦
這裡要理解概念即可。
（此為進階內容，中級只需了解概念）

### 3.5 異常值（Outliers）
`異常值（Outliers）` 是明顯偏離多數資料分布的值。
先問兩件事：
- 它是真實但極端的案例？
- 還是錄入錯誤？
常見例子：
- 真實高消費客戶
- 年齡填成 350
- 金額多打一個 0
🗣️ 白話說明：朋友說房租 800 元，你會先問是不是少打一個 0，而不是直接拿去當真實行情。
🔥🔥 必考：異常值不是看到就刪，要先判斷來源。

### 3.6 重複值（Duplicates）與格式統一（Format Consistency）
`重複值（Duplicates）` 會讓某些樣本被不合理放大。
`格式統一（Format Consistency）` 則是把日期、單位、拼法等整理一致。
常見問題：
- 同一會員重複匯入
- 訂單被寫兩次
- 日期格式混亂
- 單位不同卻混在一起
🗣️ 白話說明：把「已出貨」「出貨中」「已寄出」當三個不同狀態，系統就會很難學。

### 3.7 尺度調整（Scaling）🔥🔥
`尺度調整（Scaling）` 是把不同欄位調到更合理、可比較的尺度。
當特徵尺度差很多時，部分模型會被大數字欄位主導。

#### 3.7.1 標準化（Standardization, Z-score）
`標準化（Standardization, Z-score）` 會讓特徵落在平均值接近 0、標準差接近 1 的尺度。
常見情境：
- 特徵單位差很多
- 分布沒有固定上下界
- 模型對距離或邊界敏感
常見搭配：
- `支援向量機（Support Vector Machine, SVM）`
- `k 近鄰（k-Nearest Neighbors, k-NN）`
🗣️ 白話說明：像把不同部門績效先轉成同一評分基準，才能公平比較。

#### 3.7.2 正規化（Normalization, Min-Max Scaling）
`正規化（Normalization, Min-Max Scaling）` 常指把數值壓到固定區間，例如 0 到 1。
適合：
- 想讓欄位落在固定範圍
- 上下界相對明確
🗣️ 白話說明：像把不同科成績都換成百分比，方便一起看。

#### 3.7.3 哪些模型比較在意尺度
較在意：
- SVM
- k-NN
相對不那麼依賴：
- 決策樹（Decision Tree）
- 隨機森林（Random Forest）
```text
特徵 A：收入 30,000 ~ 2,000,000
特徵 B：是否訂閱 0 / 1
若不縮放：A 影響 >>> B
若先縮放：兩者較能公平比較
```
🔥🔥 高頻關鍵字：
- 特徵尺度差異很大
- k-NN
- SVM

#### 3.7.4 先切分，再縮放
```text
正確：
全部資料
  ↓
先切 Train / Validation / Test
  ↓
只用 Training Set 學縮放規則
  ↓
同規則套到 Validation / Test

錯誤：
全資料一起算平均 / 最大最小
  ↓
再切分
  ↓
Data Leakage
```

### 3.8 類別編碼（Categorical Encoding）🔥🔥
`類別編碼（Categorical Encoding）` 是把文字或類別欄位轉成模型可使用的形式。
常見方法：
- 標籤編碼（Label Encoding）
- One-Hot 編碼（One-Hot Encoding）
- 序位編碼（Ordinal Encoding）
- 目標編碼（Target Encoding）

#### 3.8.1 標籤編碼（Label Encoding）
把類別值直接映射成整數。
風險：
- 容易引入不存在的大小順序
🗣️ 白話說明：把 7-11、全家、萊爾富編成 1、2、3，不代表它們真的有大小關係。

#### 3.8.2 One-Hot 編碼（One-Hot Encoding）
每個類別拆成一個欄位，出現記 1，否則記 0。
優點：
- 不引入假順序
- 直觀、常見
缺點：
- 類別很多時維度暴增
- 形成高維稀疏（High-Dimensional Sparse）資料
🔥🔥 高頻：無順序、類別不多時，常優先想到 One-Hot。

#### 3.8.3 序位編碼（Ordinal Encoding）
適用於本來就有順序的類別。
例如：
- 滿意度：低 / 中 / 高
- 教育程度：高中 / 大學 / 研究所

#### 3.8.4 目標編碼（Target Encoding）
用類別對應的目標統計量表示該類別。
優點：
- 高基數欄位時可降維
風險：
- 容易造成標籤洩漏（Target Leakage）
🔥🔥 必記：若在切分前用全資料算目標編碼，答案可能被偷偷帶進特徵。
（此為進階內容，中級只需了解概念）

#### 3.8.5 ASCII 圖：Feature Encoding Decision Tree
```text
[類別欄位]
  ↓
有自然順序？
  ├─ 有 → 序位編碼（Ordinal Encoding）
  └─ 沒有
       ↓
     類別數量多嗎？
       ├─ 不多 → One-Hot 編碼（One-Hot Encoding）
       └─ 很多
            ↓
          擔心維度暴增？
            ├─ 是 → 可考慮目標編碼（Target Encoding）
            └─ 否 → One-Hot 仍可，但成本較高
陷阱：無順序類別直接做 Label Encoding
```

### 3.9 特徵工程（Feature Engineering）🔥🔥
`特徵工程（Feature Engineering）` 是把原始資料轉成模型更容易學習的表示方式。
重點不是「變複雜」，而是「變有用」。

#### 3.9.1 特徵選擇（Feature Selection）
從既有欄位挑出重要的部分。
目的：
- 降低雜訊
- 降低維度
- 提升可解釋性
- 避免過度擬合（Overfitting）
🗣️ 白話說明：履歷不會把每件小事都塞上去，挑重點反而比較有力。

#### 3.9.2 特徵轉換（Feature Transformation）
把欄位轉成更適合模型處理的形式。
例子：
- 數值縮放
- 類別編碼
- 日期拆成年、月、日、星期
- 偏態分布做對數轉換（Log Transformation）
🗣️ 白話說明：像把刷卡明細整理成「每月平均支出」，比原始流水帳更有分析價值。

#### 3.9.3 特徵建立（Feature Creation）
從原始欄位衍生新欄位。
常見：
- 近 30 天總消費
- 距離上次購買幾天
- 點擊率、轉換率
🗣️ 白話說明：YouTube 不只看你有沒有點影片，也會看你看多久、多久回來一次，很多都是衍生特徵。

#### 3.9.4 特徵工程不是欄位越多越好
過多欄位可能帶來：
- 噪音增加
- 維度膨脹
- 成本上升
- 過度擬合風險增加
🔥🔥 必記：好的特徵工程重質不重量。

### 3.10 資料切分（Dataset Split）🔥🔥
`資料切分（Dataset Split）` 是把資料切成不同用途，避免評估作弊。
三分法：
- 訓練集（Training Set）
- 驗證集（Validation Set）
- 測試集（Test Set）

#### 3.10.1 三個集合的角色
`訓練集（Training Set）`：讓模型學習。
`驗證集（Validation Set）`：比較模型、調整設定。
`測試集（Test Set）`：最後驗收。
```text
全部資料
  ├─ Training Set   → 訓練模型
  ├─ Validation Set → 選模型 / 調設定
  └─ Test Set       → 最終確認
```
🗣️ 白話說明：平常練習像訓練集，找朋友先聽簡報像驗證集，正式上台才是測試集。
🔥🔥 必考句：
- Validation 用來選模型。
- Test 用來最後確認。

#### 3.10.2 為什麼不能一直看測試集
因為你若根據測試集表現反覆修模型，測試集就不再是未見資料。
評估會過度樂觀。

#### 3.10.3 交叉驗證（Cross-Validation）
`交叉驗證（Cross-Validation）` 是用多次切分更穩定估計泛化能力（Generalization Ability）的方法。
本課只需知道：
- 資料量不大時常用
- 有助於公平比較模型
- 不能取代最終測試集
（此為進階內容，中級只需了解概念）

#### 3.10.4 ASCII 圖：Train / Validation / Test Split Process
```text
全部資料 100%
  ↓
切分
  ├─ 訓練集 60~80%
  ├─ 驗證集 10~20%
  └─ 測試集 10~20%

流程：
Train 學模型
  ↓
Validation 選模型 / 選前處理
  ↓
定版
  ↓
Test 最後確認
```

### 3.11 類別不平衡（Class Imbalance）🔥🔥
`類別不平衡（Class Imbalance）` 指某些類別樣本遠多於其他類別。
例如：
- 詐欺只占 1%
- 流失只占 5%
- 故障事件很少
這時只看 `準確率（Accuracy）` 會很危險。

#### 3.11.1 為什麼 Accuracy 不夠
若 99% 都是正常交易，
模型全部猜正常，
Accuracy 仍可能 99%，
但它根本沒抓到詐欺。
🗣️ 白話說明：100 天裡只有 1 天異常，系統天天都說正常，也能看起來很準，但真正重要的那 1 天它完全失敗。

#### 3.11.2 面對不平衡時先想三件事
- 重新抽樣（Resampling）
- 類別權重（Class Weight）
- 評估方式不能只看 Accuracy
本課只考判斷，不考實作細節。

#### 3.11.3 ASCII 圖：Class-Imbalance Decision Matrix
```text
少數類別很重要？
  ├─ 是
  │   ├─ 想補少數樣本 → 重抽樣（Resampling）
  │   ├─ 不想動資料分布 → 類別權重（Class Weight）
  │   └─ 評估時 → 不只看 Accuracy
  └─ 否
      └─ 仍要確認是否真的符合業務目標
關鍵字：詐欺 / 流失 / 異常 / 罕見疾病 / 故障
```
🔥🔥 高頻：看到少數類別很重要，就先懷疑 accuracy 是陷阱。

### 3.12 模型選擇（Model Selection）的四個軸
`模型選擇（Model Selection）` 不是背誰最強，而是看四個軸：
1. 任務型態
2. 資料特性
3. 商業需求
4. 成本限制
🗣️ 白話說明：選手機不只看跑分，還看預算、電池、相機、維修成本。模型也是一樣。

#### 3.12.1 任務型態
- 分類（Classification）：預測類別
- 回歸（Regression）：預測連續數值
- 分群（Clustering）：找相似群體

#### 3.12.2 資料特性
要看：
- 資料量大或小
- 特徵多或少
- 線性或非線性
- 類別欄位多不多
- 是否尺度差異大
- 是否類別不平衡

#### 3.12.3 商業需求
常見需求：
- 高可解釋性（Interpretability）
- 穩定性
- 上線速度
- 效能優先

#### 3.12.4 成本限制
要考慮：
- 訓練成本（Training Cost）
- 推論成本（Inference Cost）
- 維護難度
- 資料量需求

### 3.13 常見模型優缺點 🔥🔥
以下只記考試最常出的高層判斷：
- 線性模型（Linear Model）：簡單、快、可解釋性高，但複雜非線性能力有限。
- 決策樹（Decision Tree）：好懂、好說明、對縮放較不敏感，但易過度擬合。
- 隨機森林（Random Forest）：比單棵樹穩健，適合表格資料，但較難解釋。
- 支援向量機（Support Vector Machine, SVM）：小中型資料常有效，但對尺度敏感。
- k 近鄰（k-Nearest Neighbors, k-NN）：直觀，但對尺度敏感且推論較慢。
- 神經網路（Neural Network）：表現潛力高，但資料、算力與維護成本高。
🗣️ 白話說明：像找工作不是只看公司名氣，還要看通勤、薪資、主管風格、工時。模型也一樣，沒有永遠最強，只有比較適合。
> 📖 **延伸閱讀：** 常見模型取捨
> → 詳見 [supplement-常見模型取捨.md](supplement-常見模型取捨.md)

### 3.14 ASCII 圖：Algorithm-Choice Flowchart
```text
[先看業務問題]
  ↓
是分類 / 回歸 / 分群？
  ├─ 分類 / 回歸
  │    ↓
  │  需要高可解釋性？
  │    ├─ 是 → 線性模型 / 決策樹
  │    └─ 否
  │         ↓
  │       資料量大、關係複雜？
  │         ├─ 是 → 隨機森林 / 神經網路
  │         └─ 否
  │              ↓
  │            中小型資料且尺度已處理？
  │              ├─ 是 → SVM / k-NN 可考慮
  │              └─ 否 → 先回頭做前處理或用較簡單模型
  └─ 分群
       ↓
     先確認特徵表示是否合理，再選分群方法
```

### 3.15 Dataset-Size Heuristics Table
| 資料規模直覺 | 常見判斷 | 模型思路 |
|------|------|------|
| 小型資料集 | 容易過擬合 | 線性模型、決策樹、SVM、k-NN |
| 中型資料集 | 可公平比較多種方法 | 線性模型、隨機森林、SVM |
| 大型資料集 | 複雜模式較可能出現 | 隨機森林、神經網路 |
| 高維稀疏資料 | One-Hot 後維度高 | 注意縮放、成本與模型適配 |
| 類別欄位很多 | 前處理策略影響很大 | 先處理編碼與 leakage，再談模型 |
❗ 這是 heuristics，不是死規則。題目若加上「需高解釋性」「推論受限」，答案可能改變。

### 3.16 公平比較模型的原則
`公平比較模型` 代表：
- 同切分邏輯
- 同評估標準
- 不偷看測試集
- 前處理規則一致
🗣️ 白話說明：像面試新人，A 正式面試、B 只看履歷、C 跟主管吃飯聊天，最後說誰最好，這種比較根本不公平。

### 3.17 資料洩漏（Data Leakage）是本課隱形主線 🔥🔥
`資料洩漏（Data Leakage）` 指模型意外接觸到本來不該知道的資訊。
常見情況：
- 切分前先用全資料做縮放
- 切分前先做目標編碼
- 把未來資訊放到當前特徵
- 用測試集反覆調參
結果：
- 離線成績漂亮
- 上線效果失真
🗣️ 白話說明：像考前偷看到正式考卷，分數高不代表真的會。

### 3.18 本課答題順序
```text
先看任務
  ↓
再看資料問題
  ↓
決定前處理
  ↓
最後看模型 trade-off
```
只背模型名很容易錯。
先看條件，才是中級解題方式。

## Section 4: Comparison Tables (易混淆概念)
### 4.1 標準化（Standardization） vs 正規化（Normalization）
| 概念 | 標準化（Standardization） | 正規化（Normalization） |
|------|------------|------------|
| 定義 | 轉成平均接近 0、標準差接近 1 | 壓到固定區間，常見 0 到 1 |
| 差異 | 強調相對分布位置 | 強調固定範圍 |
| 適用 | SVM、k-NN、尺度差大 | 欄位想落在固定區間 |
| 風險 | 若先用全資料算統計量會 leakage | 同樣有 leakage 風險 |

### 4.2 訓練集（Training Set） vs 驗證集（Validation Set） vs 測試集（Test Set）
| 概念 | 訓練集（Training Set） | 驗證集（Validation Set） | 測試集（Test Set） |
|------|------------|------------|------------|
| 定義 | 讓模型學習 | 選模型、調設定 | 最終驗收 |
| 差異 | 直接參與訓練 | 影響模型選擇 | 不應反覆參與選擇 |
| 風險 | 資料不足會學不好 | 可能過擬合驗證集 | 一直偷看就失去公正性 |

### 4.3 One-Hot 編碼（One-Hot Encoding） vs 標籤編碼（Label Encoding）
| 概念 | One-Hot 編碼（One-Hot Encoding） | 標籤編碼（Label Encoding） |
|------|------------|------------|
| 定義 | 每類別拆成獨立欄位 | 每類別映射成整數 |
| 差異 | 不引入假順序 | 容易引入假順序 |
| 適用 | 無順序、類別數不多 | 代號表示時要小心，多半不優先 |
| 風險 | 高基數時維度暴增 | 模型誤以為有大小關係 |

### 4.4 One-Hot 編碼（One-Hot Encoding） vs 目標編碼（Target Encoding）
| 概念 | One-Hot 編碼（One-Hot Encoding） | 目標編碼（Target Encoding） |
|------|------------|------------|
| 定義 | 每類別獨立欄位 | 以目標統計量表示類別 |
| 差異 | 安全直觀但維度高 | 降維但有 leakage 風險 |
| 適用 | 類別數量有限 | 高基數欄位 |
| 考點 | 入門標準答案常是它 | 中級會考最大風險是 target leakage |

### 4.5 決策樹（Decision Tree） vs 隨機森林（Random Forest）
| 概念 | 決策樹（Decision Tree） | 隨機森林（Random Forest） |
|------|------------|------------|
| 定義 | 單一樹狀規則模型 | 多棵樹的集成方法 |
| 差異 | 易解釋但不穩定 | 較穩健但較難解釋 |
| 適用 | 高可解釋性場景 | 效果與穩健性優先 |
| 風險 | 易過擬合 | 成本較高 |

### 4.6 SVM vs k-NN
| 概念 | 支援向量機（Support Vector Machine, SVM） | k 近鄰（k-Nearest Neighbors, k-NN） |
|------|------------|------------|
| 定義 | 找分類邊界 | 看最近鄰居決定結果 |
| 差異 | 著重邊界 | 著重距離 |
| 共通點 | 都對尺度敏感 | 都對尺度敏感 |
| 風險 | 大資料成本高 | 推論成本高 |

### 4.7 線性模型（Linear Model） vs 神經網路（Neural Network）
| 概念 | 線性模型（Linear Model） | 神經網路（Neural Network） |
|------|------------|------------|
| 定義 | 結構較簡單、可解釋性高 | 可學複雜非線性關係 |
| 差異 | 快、穩、好說明 | 強、複雜、成本高 |
| 適用 | baseline、商業說明 | 大資料、複雜模式 |
| 風險 | 難處理複雜關係 | 黑盒感強、成本高 |

### 4.8 Accuracy vs 類別不平衡下的正確判斷
| 概念 | 只看 Accuracy | 類別不平衡下的正確判斷 |
|------|------------|------------|
| 核心 | 看整體猜中比例 | 看少數類別是否被抓到 |
| 問題 | 易被多數類別誤導 | 更貼近真實業務損失 |
| 適用 | 類別較平衡 | 詐欺、流失、異常、罕見事件 |
| 考點 | 常是錯誤選項 | 常是正確思考方向 |

## Section 5: 口訣 / Mnemonics
### 5.1 數據收集口訣
**「任、代、標、法」**
- 任：任務先確認
- 代：代表性要夠
- 標：標籤品質要穩
- 法：合法授權要查

### 5.2 清洗與預處理口訣
**「缺、異、重、格、縮、編」**
- 缺：缺失值
- 異：異常值
- 重：重複值
- 格：格式統一
- 縮：縮放
- 編：編碼

### 5.3 尺度調整口訣
**「差很大，先縮它」**
看到：
- 尺度差很多
- SVM
- k-NN
就先想到縮放。

### 5.4 類別編碼口訣
**「有序用序位，無序先 One-Hot，高基數防洩漏」**
- 有順序 → 序位編碼（Ordinal Encoding）
- 無順序、類別不多 → One-Hot 編碼（One-Hot Encoding）
- 類別很多 → 可想目標編碼（Target Encoding），但要防 leakage

### 5.5 資料切分口訣
**「訓練學、驗證選、測試判」**
- 訓練集學模型
- 驗證集選模型
- 測試集做最後判定

### 5.6 類別不平衡口訣
**「少數重要，別看整體」**
題目若看到：
- 詐欺
- 流失
- 異常
- 罕見疾病
就先懷疑 accuracy 是陷阱。

### 5.7 模型選擇口訣
**「任資商成」**
- 任：任務型態
- 資：資料特性
- 商：商業需求
- 成：成本限制

### 5.8 常見模型記憶法
**「線快可解、樹懂易爆、森穩較黑、S 尺度敏、K 近鄰慢、網強但貴」**
- 線：線性模型快且好解釋
- 樹：決策樹易懂但易過擬合
- 森：隨機森林穩健但較難解釋
- S：SVM 對尺度敏感
- K：k-NN 推論慢
- 網：神經網路強但成本高

## Section 6: 考試陷阱 (Exam Traps)
❌ 陷阱：資料越多越好。  
✅ 正解：更重要的是任務相關性、代表性、標籤品質與合法性。

❌ 陷阱：缺失值一律刪掉。  
✅ 正解：要看缺失比例、資料珍貴程度與是否影響代表性。

❌ 陷阱：異常值一定要刪。  
✅ 正解：先分辨是真實極端案例還是資料錯誤。

❌ 陷阱：所有模型都一定要縮放。  
✅ 正解：SVM、k-NN 通常更需要；樹模型相對不依賴，但不代表不用做其他前處理。

❌ 陷阱：標準化（Standardization）和正規化（Normalization）一樣。  
✅ 正解：兩者都是尺度調整，但目的不同；題目常用情境分辨。

❌ 陷阱：類別欄位一律用標籤編碼（Label Encoding）。  
✅ 正解：若沒有自然順序，整數代號可能引入假順序；無序類別通常先想到 One-Hot。

❌ 陷阱：One-Hot 編碼（One-Hot Encoding）永遠最好。  
✅ 正解：高基數欄位會造成維度暴增；要一起考慮成本。

❌ 陷阱：目標編碼（Target Encoding）很強，所以先做一定比較好。  
✅ 正解：它最需要小心的就是 target leakage，尤其在切分前使用時。

❌ 陷阱：驗證集（Validation Set）和測試集（Test Set）差不多。  
✅ 正解：Validation 用來選模型，Test 用來最終驗收。

❌ 陷阱：測試集成績不好，可以拿來再調一點。  
✅ 正解：一旦反覆拿測試集調整，它就失去公正性。

❌ 陷阱：Accuracy 很高就代表模型很好。  
✅ 正解：在類別不平衡情境下，accuracy 可能嚴重誤導。

❌ 陷阱：決策樹（Decision Tree）一定比隨機森林（Random Forest）差。  
✅ 正解：若需要高可解釋性，決策樹反而可能更合適。

❌ 陷阱：神經網路（Neural Network）最強，所以優先選它。  
✅ 正解：中級考的是 trade-off。資料量、解釋性、成本都會改變答案。

❌ 陷阱：欄位越多越完整。  
✅ 正解：欄位過多可能增加噪音、成本與過擬合風險。

❌ 陷阱：交叉驗證（Cross-Validation）做了，就不需要測試集。  
✅ 正解：交叉驗證有助比較，但最終仍應有獨立測試集。

❌ 陷阱：前處理先在全資料上一起做比較省事。  
✅ 正解：這是典型資料洩漏（Data Leakage）來源；正確順序是先切分，再由訓練集學轉換規則。

❌ 陷阱：樹模型不看尺度，所以前處理都能省。  
✅ 正解：樹模型只是相對不依賴縮放，仍要處理缺失值、重複值、格式與編碼問題。

❌ 陷阱：模型選擇只看演算法特性，不看業務條件。  
✅ 正解：本課重點就是 trade-off，要把可解釋性、時效、成本一起看。

## Section 7: 情境題快速判斷 (Scenario Quick-Judge)
🔑 看到關鍵字 → 選這個答案
- 特徵尺度差異很大 → 先想到標準化（Standardization）或正規化（Normalization）
- `SVM`、`k-NN` → 先警覺尺度敏感
- 決策樹（Decision Tree）、隨機森林（Random Forest） → 縮放通常不是第一優先考點
- 類別欄位無自然順序、類別不多 → One-Hot 編碼（One-Hot Encoding）
- 類別欄位有自然順序 → 序位編碼（Ordinal Encoding）
- 高基數類別欄位 → 小心 One-Hot 維度暴增，可考慮目標編碼（Target Encoding），但注意 leakage
- 切分前先做目標編碼 → 資料洩漏（Data Leakage）
- 切分前先用全資料算縮放統計量 → 資料洩漏（Data Leakage）
- 驗證不同模型哪個較好 → 驗證集（Validation Set）
- 最後確認模型泛化能力 → 測試集（Test Set）
- 用測試集一直調模型 → 評估不公、測試集失效
- 詐欺、流失、異常、罕見疾病 → 類別不平衡（Class Imbalance），別只看 Accuracy
- 少數類別很重要 → 想到重抽樣（Resampling）、類別權重（Class Weight）、評估不能只看 Accuracy
- 欄位很多、資訊重複、模型易過擬合 → 特徵選擇（Feature Selection）
- 日期拆成年月日星期 → 特徵轉換（Feature Transformation）
- 用交易紀錄算近 30 天總消費 → 特徵建立（Feature Creation）
- 需要高可解釋性 → 線性模型（Linear Model）或決策樹（Decision Tree）
- 表格資料、想兼顧穩健與效果 → 隨機森林（Random Forest）
- 小中型資料、分類問題、尺度已處理 → SVM 可考慮
- 小型資料、想要直觀基準法 → k-NN 可考慮
- 資料量大、模式複雜、成本可接受 → 神經網路（Neural Network）
- 題目強調「不是越複雜越好」 → 模型選擇要看 trade-off
- 題目問「公平比較模型」 → 同切分、同標準、不可偷看測試集
