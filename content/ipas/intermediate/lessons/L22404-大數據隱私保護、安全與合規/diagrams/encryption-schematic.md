# 加密類型示意圖 (Encryption Schematic)

## 靜態加密 vs 傳輸加密 vs 同態加密

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    三種加密情境對照                                        │
└─────────────────────────────────────────────────────────────────────────┘

①  靜態加密（Encryption at Rest）
   ─────────────────────────────
   資料庫 / 硬碟 / 備份

   [原始資料] ──AES-256 加密──► [🔒密文儲存]

   ▸ 保護：資料被實體竊取時無法讀取
   ▸ 演算法：AES-256（對稱加密，最常見）
   ▸ 考試關鍵：「資料儲存」場景 → 選靜態加密


②  傳輸加密（Encryption in Transit）
   ──────────────────────────────────
   用戶端           網路             伺服器

   [原始資料] ──TLS 1.3 握手──► [🔒密文傳輸] ──解密──► [原始資料]
                  🔑
              (公私鑰交換)
   ▸ 保護：資料在傳輸過程中被竊聽
   ▸ 協定：TLS 1.3（HTTPS 底層）
   ▸ 考試關鍵：「資料傳輸 / 網路通訊」場景 → 選傳輸加密


③  同態加密（Homomorphic Encryption）
   ────────────────────────────────
         ⭐ 114年第二梯次官方考題出現 ⭐

   [原始資料] ──加密──► [🔒密文]
                              │
                              ▼  無須解密直接運算！
                         [🔒密文計算結果]
                              │
                              ▼ 解密
                         [正確的運算結果]

   ▸ 保護：運算過程中資料不暴露（連運算方也看不到明文）
   ▸ 典型應用：雲端加密計算、醫療資料外包分析
   ▸ 考試關鍵：「加密狀態下仍需計算」→ 唯一答案是同態加密
   ▸ ⚠️ 不是傳輸加密！不是靜態加密！是「計算加密」


┌──────────────────────────────────────────────────────────────────┐
│ 三者速查表                                                          │
├──────────────────┬───────────────────┬──────────────────────────┤
│ 加密類型          │ 保護時機           │ 關鍵演算法 / 協定          │
├──────────────────┼───────────────────┼──────────────────────────┤
│ 靜態（At Rest）  │ 儲存中（磁碟/備份） │ AES-256                  │
│ 傳輸（In Transit)│ 傳輸中（網路）      │ TLS 1.3                  │
│ 同態（Homomorphic│ 計算中（雲端運算）  │ Paillier / BFV / CKKS    │
└──────────────────┴───────────────────┴──────────────────────────┘
```

## 存取控制模型

```
RBAC（Role-Based Access Control）
────────────────────────────────
  用戶 → 角色 → 權限

  小明（資料工程師）→ [Engineer 角色] → 讀取資料、執行 ETL
  小美（分析師）    → [Analyst 角色]  → 讀取報表、查詢聚合統計
  小偉（DBA）       → [DBA 角色]      → 讀寫所有表、管理用戶

  ✅ 優點：管理簡單，角色固定
  ❌ 限制：無法細緻控制（如「只能看自己部門的資料」）


ABAC（Attribute-Based Access Control）
─────────────────────────────────────
  用戶屬性 + 資源屬性 + 環境條件 → 動態決策

  小明（部門=財務, 職級=P4）
    ├─ 存取「財務報表（部門=財務）」→ ✅ 允許（部門相符）
    ├─ 存取「薪資資料（職級≥P5）」 → ❌ 拒絕（職級不足）
    └─ 存取「任何資料（時間=凌晨3點）」→ ❌ 拒絕（時間異常）

  ✅ 優點：細緻靈活，支援動態條件
  ❌ 限制：設定複雜，效能略低


欄位層級安全（Column-Level Security）
──────────────────────────────────
  同一張表，不同欄位有不同存取權限：

  用戶表：
  ┌───────┬──────┬────────┬──────────┐
  │ user_id│ name │ salary │ ssn      │
  ├───────┼──────┼────────┼──────────┤
  │   1   │ 可見  │  🔒    │    🔒    │  ← 一般員工：只看 name
  │   2   │ 可見  │  🔒    │    🔒    │  ← HR：看 salary
  │   3   │ 可見  │  可見  │    🔒    │  ← 法務：看 ssn
  └───────┴──────┴────────┴──────────┘
```
