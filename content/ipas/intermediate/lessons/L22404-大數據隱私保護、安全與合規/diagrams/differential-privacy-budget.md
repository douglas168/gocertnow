# 差分隱私預算圖 (Differential Privacy Budget)

## ε (Epsilon) 直覺圖

```
隱私保護強度
    ▲
    │  ε = 0.01  ████████████████████  非常私密（幾乎無資訊洩露）
    │  ε = 0.1   ████████████████      很私密
    │  ε = 1.0   ████████████          適中（學術研究常用值）
    │  ε = 5.0   ██████                較低（商業分析常用）
    │  ε = ∞     ██                    無保護（等同公開資料）
    │
    └────────────────────────────────────────────► ε 值越大 → 噪聲越少 → 資訊越多 → 隱私越弱
```

## 隱私預算組合定理

```
情境：你的資料庫要被查詢 3 次，每次隱私預算為 ε₁, ε₂, ε₃

┌──────────────────────────────────────────────────────────┐
│ 順序組合（Sequential Composition）                        │
│                                                          │
│  查詢1(ε₁) → 查詢2(ε₂) → 查詢3(ε₃)                     │
│                                                          │
│  總隱私損失 = ε₁ + ε₂ + ε₃                              │
│                                                          │
│  例：ε₁=0.5, ε₂=0.5, ε₃=1.0 → 總計 ε = 2.0            │
│  ⚠️  每新增一次查詢，隱私損失累積！                        │
└──────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────┐
│ 平行組合（Parallel Composition）                          │
│                                                          │
│  資料分為不重疊子集：                                     │
│  子集A ──查詢1(ε₁)──►                                   │
│  子集B ──查詢2(ε₂)──►  總隱私損失 = max(ε₁, ε₂, ε₃)    │
│  子集C ──查詢3(ε₃)──►                                   │
│                                                          │
│  例：ε₁=0.5, ε₂=0.8, ε₃=1.0 → 總計 ε = 1.0 (取最大)  │
│  ✅ 不重疊子集→隱私損失不累加，取最大值即可！              │
└──────────────────────────────────────────────────────────┘

🔥🔥 考試陷阱：平行組合「不是加總」而是「取最大值」
```

## Laplace 機制 vs Gaussian 機制

```
查詢類型          │ 使用機制         │ 雜訊分佈        │ 典型應用
─────────────────┼─────────────────┼────────────────┼─────────────
純量查詢          │ Laplace 機制     │ Laplace(0,b)   │ 統計計數、平均值
(Numeric scalar) │                 │ b = Δf/ε       │ 「這群人的平均年齡」
─────────────────┼─────────────────┼────────────────┼─────────────
向量查詢          │ Gaussian 機制    │ N(0,σ²)        │ 嵌入向量、模型梯度
(Vector)         │                 │ σ = Δf·√(2ln   │ 「推薦系統特徵向量」
                 │                 │    (1.25/δ))/ε │
─────────────────┼─────────────────┼────────────────┼─────────────

Δf = 全局靈敏度（Global Sensitivity）= 一個人的資料最多影響查詢結果多少
```

## ε-DP 預算算術範例（考試常見題型）

```
題目：某公司對醫療資料庫做以下查詢：
  - 查詢1：統計病患數（ε=1.0）
  - 查詢2：計算平均年齡（ε=0.5）
  - 查詢3：統計特定疾病人數（ε=0.5）

問：若三個查詢在同一資料集上依序執行，總隱私預算消耗為何？

答：順序組合 → ε_total = 1.0 + 0.5 + 0.5 = 2.0

進階問：若將資料集分為三個不重疊子集，各查詢對應一個子集，總預算為何？

答：平行組合 → ε_total = max(1.0, 0.5, 0.5) = 1.0  ✅ 更省預算！
```
