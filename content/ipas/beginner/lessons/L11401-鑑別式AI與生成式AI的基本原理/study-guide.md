## 3. 核心概念（Core Concepts）

### 3.1 鑑別式AI基本原理（Discriminative AI Fundamentals）

`鑑別式AI（Discriminative AI）` 的核心任務就一句話：**拿到一筆資料，告訴你它「是什麼」或「會怎樣」。**

它不創造任何新東西——只負責**分辨、分類、預測**。你給它一封信，它告訴你是不是垃圾郵件。你給它一張 X 光片，它告訴你可能是什麼疾病。你給它一筆交易紀錄，它告訴你有沒有詐騙嫌疑。

🗣️ **白話說明：** 把鑑別式AI想成 7-11 的自助結帳機器。你掃一罐飲料的條碼，它只做一件事——辨識出「這是什麼商品、多少錢」。它不會幫你調一杯新的飲料，它只是告訴你手上這杯是什麼。

#### 🔥 鑑別式AI的特性

| 特性 | 說明 |
|------|------|
| 核心任務 | 分類（Classification）、預測/迴歸（Prediction/Regression），也常用於異常偵測（Anomaly Detection） |
| 資料需求 | 通常需要**有標註的資料**（Labeled Data）——每筆資料都要附正確答案 |
| 學習方式 | `監督式學習（Supervised Learning）`——你在 L11302 已經學過 |
| 輸出結果 | 類別標籤（「是 / 不是」）或數值預測（「這房子可能值 1,200 萬」） |
| 學習邏輯 | 找到最佳的`決策邊界（Decision Boundary）`——就是在不同類別之間畫一條分界線 |

> **連結 L11302：** 你在上一課學的邏輯迴歸、SVM、決策樹、隨機森林——全部都是鑑別式模型。當時你學的「監督式學習」就是鑑別式AI最主要的學習方式。

#### 🔥🔥 鑑別式AI的典型應用場景

記住這些場景，考試情境題直接用：

| 應用場景 | 對應的鑑別式任務 |
|---------|----------------|
| `垃圾郵件偵測（Spam Detection）` | 分類：這封信是「垃圾」還是「正常」 |
| `詐騙偵測（Fraud Detection）` | 異常偵測：這筆交易是否異常 |
| `醫療診斷（Medical Diagnosis）` | 分類：X 光片判斷疾病類別 |
| `產品瑕疵檢測（Defect Detection）` | 分類：良品 vs 瑕疵品 |
| `情感分析（Sentiment Analysis）` | 分類：這則評論是正面/負面/中性 |
| `光學字元辨識（OCR）` | 分類：這個像素區域是哪個字 |
| `人臉辨識（Face Recognition）` | 分類：這張臉是誰 |
| `車牌辨識` | 分類：這塊區域的文字是什麼 |

🗣️ **白話說明：** 想像蝦皮的賣家評價系統。顧客留一則評論「出貨速度快但包裝有點爛」——鑑別式AI就像一個助理，讀完以後告訴你：「這則評論整體偏正面 (0.65 分)」。它只做判斷，不會幫你寫回覆。

#### 🔥 常見鑑別式模型（考試辨認用）

考試會給你模型名稱，要你判斷屬於鑑別式還是生成式。以下必須記住是鑑別式：

- `邏輯迴歸（Logistic Regression）`
- `支援向量機（Support Vector Machine, SVM）` ⚠️ 高頻陷阱題！
- `決策樹 / 隨機森林（Decision Tree / Random Forest）`
- `梯度提升機（Gradient Boosting）`
- 用於分類任務的`神經網路（Neural Networks）`

```
鑑別式AI的工作流程（概念圖）

  輸入資料        鑑別式模型         輸出結果
  ┌────────┐    ┌──────────┐    ┌──────────┐
  │ 一封信  │───>│ 學到的   │───>│ 垃圾郵件  │
  │ 一張圖  │    │ 決策邊界  │    │ 正常郵件  │
  │ 一筆交易│    │          │    │ 數值預測  │
  └────────┘    └──────────┘    └──────────┘

        ※ 只做「判斷」，不創造新東西
```

### 3.2 生成式AI基本原理（Generative AI Fundamentals）

`生成式AI（Generative AI）` 的核心任務也是一句話：**學習資料長什麼樣子，然後創造出全新的、以前不存在的內容。**

它不是在做判斷——它是在「創作」。你給它一段提示詞（prompt），它幫你寫文章、畫圖、寫程式、作曲、甚至生成影片。

🗣️ **白話說明：** 如果鑑別式AI是 7-11 的條碼掃描機（告訴你這是什麼），生成式AI就是隔壁的手搖飲料師傅。你跟師傅說「我要一杯半糖少冰的芒果鮮奶茶」，師傅根據自己對所有飲料的「理解」，現場調配出一杯全新的飲料——這杯飲料之前不存在，是他「生成」出來的。

#### 🔥 生成式AI的特性

| 特性 | 說明 |
|------|------|
| 核心任務 | 生成（Generation）全新的文字、圖像、音頻、影片、程式碼、3D模型 |
| 資料需求 | 可以使用**未標註的資料**（Unlabeled Data）——大量資料丟進去就能學 |
| 學習方式 | 可使用`非監督式學習（Unsupervised Learning）`——不一定需要人工標註 |
| 輸出結果 | 全新的、以前不存在的內容 |
| 學習邏輯 | 學習`資料分布（Data Distribution）`——理解「這類資料整體長什麼樣」 |

#### 🔥🔥 生成式AI的典型應用場景

| 應用場景 | 生成的內容類型 | 代表工具 |
|---------|--------------|---------|
| `文本生成（Text Generation）` | 文章、摘要、翻譯、對話 | ChatGPT、Claude |
| `圖像生成（Image Generation）` | 插圖、設計稿、修圖 | DALL-E、Midjourney、Stable Diffusion |
| `程式碼生成（Code Generation）` | 程式碼、單元測試 | GitHub Copilot |
| `音頻/音樂生成（Audio Generation）` | 配樂、語音合成 | Suno、ElevenLabs |
| `影片生成（Video Generation）` | 短影片、動畫 | Sora |
| `3D模型生成` | 建築、遊戲素材 | — |

🗣️ **白話說明：** 你用 IG 的限時動態，打一段文字讓 AI 幫你自動生成配圖——那就是生成式AI在工作。它「看過」幾百萬張圖片後，理解了「圖片應該長什麼樣」，於是能根據你的描述「畫」出一張全新的圖。

#### 🔥 常見生成式模型（考試辨認用）

以下必須記住是生成式，尤其前兩個是高頻陷阱：

- `生成對抗網路（Generative Adversarial Network, GAN）` ⚠️ 名字裡就有 Generative！
- `變分自動編碼器（Variational Autoencoder, VAE）` ⚠️ 容易被誤認為鑑別式
- `擴散模型（Diffusion Model）` — Stable Diffusion 用的就是這個
- `自迴歸模型（Autoregressive Models）` — GPT 系列的基礎

> ⚠️ GAN、VAE、Diffusion Model 的內部架構（generator/discriminator、encoder/decoder 等）是中級內容，初級只需能辨認它們**屬於生成式**即可。（此為中級內容，初級只需了解概念）

### 3.3 技術測試與驗證 + 模型目的與特性

不管是鑑別式還是生成式AI，上線前都必須經過`技術測試與驗證（Technical Testing & Validation）`。這一段在考試中常搭配情境題出現——給你一個場景，問你應該用什麼驗證方法或評估指標。

#### 🔥 概念驗證（Proof of Concept, PoC）

導入AI之前，先做小規模實驗確認「這個方向走得通」再投入大量資源。

🗣️ **白話說明：** 你想在學校附近開一家飲料店，不會一開始就砸 200 萬裝潢——你會先在校門口擺攤賣三天，看看有沒有人買。這就是 PoC。企業導入AI也一樣，先用少量資料、簡單模型測試，驗證效果後再擴大。

```
PoC 的流程概念

  有個 AI 點子 ──> 小規模測試 ──> 可行？
                                   │
                            ┌──────┴──────┐
                            ▼              ▼
                          是 → 擴大投入    否 → 調整方向或放棄
                                          （省下大筆冤枉錢）
```

#### 🔥 偏差-變異權衡（Bias-Variance Tradeoff）

這個概念你在 L11301 的過度擬合/擬合不足已經碰過。換一個角度重新理解：

- `偏差（Bias）`：模型太簡單、抓不到重點 → `擬合不足（Underfitting）`
- `變異（Variance）`：模型太複雜、把雜訊也背下來 → `過度擬合（Overfitting）`
- 好的模型 = 在偏差和變異之間取得**平衡**

🗣️ **白話說明：** 準備考試的時候——

- **偏差太高（Underfitting）**= 你只讀了課本目錄就去考試，連重點都沒抓到
- **變異太高（Overfitting）**= 你把考古題的每一個選項連號碼都背下來，結果換個出題方式就全錯
- **剛好**= 你讀懂了觀念，題目怎麼變都能應對

> ⚠️ 偏差-變異權衡在初級只考概念，不會考數學公式。能用白話說明什麼是「太簡單」和「太複雜」就夠了。

#### 🔥🔥 評估指標選擇——不平衡資料該看什麼？

考試超愛考這個情境：**當資料極端不平衡時（如醫療漏診、詐騙偵測），應該優先看哪個指標？**

關鍵在於**誤判的代價不對稱**——你要問自己：「漏掉（False Negative）比較嚴重，還是誤報（False Positive）比較嚴重？」

- `召回率（Recall）`：所有真正陽性的裡面，模型抓到了幾個？→ **漏掉的代價高時優先看這個**
- `精確率（Precision）`：模型說是陽性的裡面，真正是陽性的有幾個？→ **誤報的代價高時優先看這個**

🗣️ **白話說明：**

**場景一（優先 Recall）：** 你是醫院的 AI 系統，負責從 X 光片中篩出疑似肺癌的病患。你寧可**多抓幾個健康的人去做進一步檢查**（他們頂多花點時間），也不能**漏掉一個真正有肺癌的病人**（他錯過黃金治療期可能喪命）。→ 漏掉的代價 >> 誤報的代價 → 用 Recall。

**場景二（優先 Precision）：** 你是垃圾郵件過濾器。如果你把一封重要的客戶郵件誤判為垃圾郵件，業務可能因此丟失訂單。→ 誤報的代價 >> 漏掉的代價 → 用 Precision。

**共同陷阱：** 這兩種場景用正確率（Accuracy）反而會被騙——如果 1,000 人裡只有 5 人有病，你全部猜「沒病」也有 99.5% 正確率，但 5 個病人全漏了。

> ⚠️ 初級不考 Confusion Matrix 的計算，但你要知道「不平衡資料 → 先問誤判代價 → Recall 或 Precision 比 Accuracy 重要」這個判斷邏輯。考試最常考的情境是醫療漏診和詐騙偵測，這兩個都是 **Recall 優先**。

#### 監理沙盒（Regulatory Sandbox）

`監理沙盒（Regulatory Sandbox）` 是政府提供的「安全實驗區」，讓企業在受控環境中測試具有潛在風險的 AI 創新應用，不需要事先取得所有法規許可。

🗣️ **白話說明：** 就像新手駕駛可以先在駕訓場練車，不用直接上高速公路。AI 監理沙盒讓企業可以在「有圍欄的場地」裡測試新技術——出了問題也不會真的傷到消費者。

> 💡 台灣《人工智慧基本法》已於 2026 年 1 月 14 日施行，明訂七大基本原則。AI 治理相關的考試重點已在 **L11102 AI治理概念** 詳細說明，這裡只需記住：台灣已有 AI 專法，且監理沙盒是其中鼓勵創新的重要機制之一。

```
  │  目的：判斷/分類     │    │  目的：創造新內容    │
  │                     │    │                     │
  │  特性：              │    │  特性：              │
  │  • 需要標註資料      │    │  • 可用未標註資料    │
  │  • 輸出是標籤/數值   │    │  • 輸出是新內容      │
  │  • 評估方式明確      │    │  • 評估較主觀        │
  │   (正確率/召回率)    │    │   (人類偏好/品質)    │
  │  • 決策可解釋性較高  │    │  • 可能產生幻覺      │
  │                     │    │   (Hallucination)    │
  └─────────────────────┘    └─────────────────────┘
```

### 3.4 模型部署與效能管理

模型訓練好之後不是結束——它必須被「部署」到真實環境中才能發揮作用。你在 L11301 學的 ML 工作流程最後一步就是「部署與監控」，這堂課把這個概念展開。

#### 🔥🔥 模型部署（Model Deployment）的三種方式 + 混合架構

`模型部署（Model Deployment）` 就是把訓練好的模型放到可以實際處理資料的地方。有三大類部署方式，加上一個組合拳：

**1. 雲端部署（Cloud Deployment）**

把模型放在雲端伺服器（如 AWS、GCP、Azure）上運行。

🗣️ **白話說明：** 就像你把照片存在 Google 雲端硬碟——資料離開你的手機，存在 Google 的機房裡。好處是空間幾乎無限，壞處是你得有網路才能存取，而且你的照片不在你手上。

**2. 邊緣部署（Edge Deployment）**

把模型直接放在使用者端的設備上運行（手機、IoT 設備、工廠機台等）。

🗣️ **白話說明：** 就像你手機的 Face ID——辨識你的臉不需要連網，模型就在你手機裡面跑。好處是超快（不用等網路往返），壞處是手機的算力有限，只能跑小模型。

**3. 內部部署（On-Premise Deployment）**

企業自己買伺服器，把模型放在公司機房裡跑。

🗣️ **白話說明：** 就像你在家裡架一台 NAS（網路儲存器）存照片——資料完全在你家，不怕外洩，但你要自己花錢買設備、自己維護。

**4. 混合架構（Hybrid Deployment）**

結合以上方式的優點。最常見的模式是：**雲端訓練模型 + 邊緣/內部做推理**。

🗣️ **白話說明：** 就像 Uber Eats 的物流模式——大數據分析（訂單預測、路線優化）在雲端處理，但外送員手機上的即時導航在本地跑。大的在雲端做，快的在邊緣做。

```
四種部署方式一覽

                算力          延遲          資料安全        成本模式
  ┌──────────┬────────────┬───────────┬──────────────┬────────────┐
  │ 雲端     │ ★★★★★     │ 較高      │ 資料離場     │ 按用量付費 │
  │ Cloud    │ 幾乎無限    │ (要傳資料) │ (需信任供應商)│            │
  ├──────────┼────────────┼───────────┼──────────────┼────────────┤
  │ 邊緣     │ ★★         │ 極低      │ 資料不離場   │ 設備成本   │
  │ Edge     │ 受設備限制   │ (本地處理) │ (高隱私)     │            │
  ├──────────┼────────────┼───────────┼──────────────┼────────────┤
  │ 內部部署 │ ★★★        │ 低        │ 完全本地化   │ 高初期投資 │
  │ On-Prem  │ 自建規模     │ (在公司內) │ (最高安全)   │ + 維護成本 │
  ├──────────┼────────────┼───────────┼──────────────┼────────────┤
  │ 混合     │ 彈性搭配     │ 依架構    │ 依架構      │ 最佳化組合 │
  │ Hybrid   │ 各取所長     │          │             │            │
  └──────────┴────────────┴───────────┴──────────────┴────────────┘
```

**🔥 部署方式選擇的判斷邏輯：**

- 需要**大量算力、高擴展性** → 雲端
- 需要**低延遲、即時反應** → 邊緣
- 需要**高安全、法規合規** → 內部部署
- 需要**兩者兼顧** → 混合架構

🗣️ **白話說明：** 想像你要處理 104 人力銀行的履歷篩選系統——

- 用雲端：因為每天有幾十萬筆履歷要處理，需要大算力
- 如果是台積電內部的員工績效預測系統？用內部部署：因為員工資料高度機密，不能離場
- 如果是工廠產線上即時偵測瑕疵品？用邊緣部署：因為每個零件通過的速度很快，不能等雲端回傳結果

> 💡 2026年考試新增了**邊緣運算 vs 即時推論**的場景判斷題。記住：需要即時決策且不能斷網的場景 → 邊緣部署。

#### 🔥 批次推論 vs 即時推論（Batch vs Real-time Inference）

模型部署後，推論方式也有兩種選擇：

- `即時推論（Real-time Inference）`：使用者每發一個請求，模型立刻回應。適合聊天機器人、即時偵測等需要低延遲的場景。
- `批次推論（Batch Inference）`：累積一批資料後一次處理。適合每天跑一次的報表預測、大量郵件分類等不要求即時回應的場景。

🗣️ **白話說明：** 即時推論就像便利商店店員——客人一來就結帳。批次推論就像郵差——不是每封信到就送，而是集中一批一起送，效率更高但不即時。

#### 🔥 效能管理（Performance Management）

模型部署後，需要持續關注三大效能指標：

**1. 延遲（Latency）**

從輸入資料到模型開始回應所經過的時間。對使用者體驗影響最大。

🗣️ **白話說明：** 就像你在 LINE 傳訊息給朋友——你按下送出到看到「已讀」之間的等待時間。延遲越低，使用者感覺系統越「快」。

> 補充：生成式AI 有兩種延遲概念——`首個 Token 產出時間（Time to First Token, TTFT）`（按下送出到第一個字出現的時間）和`每 Token 產出時間`（之後每個字跑出來的速度）。初級知道「延遲 = 等待時間」就夠了。

**2. 吞吐量（Throughput）**

單位時間內模型能處理/產出的數量（例如 token/秒）。吞吐量越高，基礎設施的投資回報越高。

🗣️ **白話說明：** 就像 Uber Eats 的出餐速度。一個廚房一小時能做 30 份便當 = 吞吐量 30。如果你的餐廳生意很好，你會希望這個數字越高越好。

**2.5 有效吞吐量（Goodput）**

`有效吞吐量（Goodput）` 是在**維持目標延遲水準**下，系統實際達成的有效產出量。

🗣️ **白話說明：** 回到 Uber Eats 的例子。那個廚房一小時做了 30 份便當（吞吐量 = 30），但其中 5 份因為出餐太慢被顧客取消、3 份因為品質太差被退貨。真正成功送到顧客手上且顧客滿意的只有 22 份——這就是 Goodput。Goodput 比單純的吞吐量更能反映系統的**真實服務品質**。

> ⚠️ 區分：吞吐量看「總共做了多少」，有效吞吐量看「在品質達標的前提下做了多少」。

**3. 成本（Cost）**

- `訓練成本（Training Cost）`：一次性的——訓練一個模型花多少錢
- `推理成本（Inference Cost）`：持續累積的——每次使用模型回答問題都要花錢

🗣️ **白話說明：** 訓練成本就像開一家飲料店的裝潢費（花一次），推理成本就像每天的食材費（持續支出）。ChatGPT 每回答你一個問題，OpenAI 就在燒 GPU 費用——這就是推理成本。

> **🔥 考試重點：** 推理成本會隨使用量持續增長，是 AI 應用長期營運的最大成本來源。不要以為「模型訓練好就不用花錢了」。

```
效能三角——三者互相制約

        延遲 (Latency)
            ▲
           / \
          /   \           想要低延遲 + 高吞吐？ → 要花更多錢
         /     \          想要省成本？ → 可能要犧牲延遲或吞吐
        /       \
       ▼─────────▼
  吞吐量          成本
  (Throughput)   (Cost)
```

#### 資料漂移與模型監控

`資料漂移（Data Drift）` 是指模型部署後，**真實世界的資料和當初訓練用的資料漸漸變得不一樣**。

🗣️ **白話說明：** 你用 2024 年的蝦皮消費資料訓練了一個推薦模型。到了 2026 年，大家的購物習慣變了（更多人買生鮮、更少人買衣服），但你的模型還活在 2024 年的世界裡——推薦的東西越來越不準。這就是資料漂移。

因此模型部署後需要持續做`模型監控（Model Monitoring）`：

- 定期檢查模型效能有沒有下降
- 設定異常告警（延遲突然變高、預測準確率下降）
- 發現資料漂移時，用新資料重新訓練或微調模型

> ⚠️ 注意區分：**技術測試與驗證**是模型「上線前」的工作（PoC、偏差-變異、評估指標選擇）；**模型監控**是模型「上線後」的持續工作（資料漂移、效能告警）。考試可能用情境題混淆這兩者。

## 5. 口訣 / Mnemonics

### 口訣一：鑑別 vs 生成——「鑑分生創」

> **「鑑」別式做「分」類，「生」成式搞「創」作**

四個字：鑑分生創。看到情境題先問自己：「這個任務是在分類/判斷，還是在創造新東西？」

### 口訣二：模型歸類——「S決邏隨 = 鑑別四天王」

> **S**VM、**決**策樹、**邏**輯迴歸、**隨**機森林 → 全部鑑別式

> **G**AN、**V**AE、**D**iffusion、G**P**T → 全部生成式（口訣：**GVDP 生四寶**）

### 口訣三：部署選擇——「算快安」

> - **算**力不夠？→ 雲端
> - 要很**快**？→ 邊緣
> - 要很**安**全？→ 內部部署

三個問題問完，部署方式就選好了。

### 口訣四：效能三指標——「延吞成」

> **延**遲看「等多久」、**吞**吐看「做多少」、**成**本看「花多少」

### 口訣五：不平衡資料——「寧可錯抓，不能漏放」

> 醫療漏診 / 詐騙偵測 → Recall（召回率）→ 寧可多抓幾個正常人，不能漏掉一個病人/罪犯

## 6. 考試陷阱（Exam Traps）

### 陷阱 1：「生成式AI只能生成文字」

❌ 生成式AI = 文字生成工具（像 ChatGPT 那樣聊天）

✅ 生成式AI能生成**文字、圖像、音頻、影片、程式碼、3D模型**等多種內容。ChatGPT 只是其中一種應用，DALL-E 生成圖像、Sora 生成影片、Copilot 生成程式碼——全部都是生成式AI。

### 陷阱 2：「鑑別式AI就是分類器」

❌ 鑑別式AI = 分類（Classification）

✅ 鑑別式AI除了分類，還能做**迴歸（Regression）**（預測房價等連續數值）和**異常偵測（Anomaly Detection）**（找出不正常的交易）。「分類」是最常見的任務，但不是唯一的。

### 陷阱 3：「模型部署就是上傳到雲端」

❌ 部署 = 放上雲端

✅ 部署有**雲端、邊緣、內部部署、混合架構**四種選擇。工廠即時瑕疵檢測用邊緣部署、銀行機密系統用內部部署——不是所有東西都適合放雲端。

### 陷阱 4：模型名稱歸類錯誤

❌ SVM 是生成式模型（因為它很強大/很常用）

✅ SVM（支援向量機）是**鑑別式**模型。它的任務是找決策邊界做分類，不會生成新內容。

❌ VAE 是鑑別式模型（因為 Autoencoder 聽起來像在「編碼」不像在「生成」）

✅ VAE（變分自動編碼器）是**生成式**模型。它的目的是學習資料分布並生成新資料。

❌ GAN 是鑑別式模型（因為名字裡有 "Adversarial" 對抗的意思）

✅ GAN（生成對抗網路）是**生成式**模型。名字第一個字 "Generative" 就告訴你了。

### 陷阱 5：混淆「測試驗證」和「部署監控」

❌ 資料漂移（Data Drift）是模型訓練時的驗證步驟

✅ 資料漂移是模型**部署後**才會出現的問題——真實世界資料和訓練資料漸漸不同。它屬於**部署後監控**的範疇，不是上線前的測試驗證。

### 陷阱 6：「Accuracy 是萬用指標」

❌ 不管什麼情況，正確率（Accuracy）越高模型越好

✅ 當資料極端不平衡時（如 1,000 人中只有 5 人有病），全猜「沒病」也有 99.5% Accuracy——但這個模型毫無用處。此時應優先看 **Recall**（召回率）。

### 陷阱 7：「推理不用花錢」

❌ 模型訓練好以後，使用就不用花什麼錢了

✅ 訓練是一次性成本，但**推理成本會隨使用量持續增長**。對於大規模 AI 應用（如 ChatGPT），推理成本才是長期最大的花費。

## 7. 情境題快速判斷（Scenario Quick-Judge）

### 表 7.1 鑑別式 vs 生成式——情境關鍵字速查

| 題目出現的關鍵字/場景 | 答案 | 原因 |

### 表 7.4 效能指標速查

| 題目出現的關鍵字/場景 | 答案 | 原因 |
