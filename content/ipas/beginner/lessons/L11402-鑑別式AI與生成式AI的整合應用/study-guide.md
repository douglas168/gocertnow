## 3. 核心概念（Core Concepts）

### 3.1 從 L11401 到 L11402：從「認識」到「應用」

你在 L11401 已經學會了最重要的一條線：**鑑別式AI = 幫你分辨，生成式AI = 幫你創造**。模型名稱（SVM、GAN、GPT 等）你也能歸類了，部署方式和效能管理的概念也有了。

這一課的核心問題只有一個：**在真實世界裡，這兩種 AI 怎麼被用、怎麼被組合？**

🗣️ **白話說明：** L11401 像是認識了刀和鍋，L11402 是學會炒出一盤菜。刀（鑑別式）切好食材、鍋（生成式）加熱烹調——真正的一道菜，往往兩個都需要。

---

### 3.2 電腦視覺（Computer Vision, CV）🔥🔥

`電腦視覺（Computer Vision, CV）` 是讓電腦「看懂」圖片或影片的 AI 技術。這是三大領域裡**鑑別式和生成式應用都最豐富**的一個，考試非常愛出。

#### 鑑別式CV應用——看懂、分辨、偵測

| 應用 | 說明 | 生活例子 |
|------|------|----------|
| `影像分類（Image Classification）` | 看一張圖，判斷它屬於哪個類別 | Google 相簿自動把你的貓照片歸到「貓」資料夾 |
| `物件偵測（Object Detection）` | 找出圖片中「每一個物件在哪裡、是什麼」 | iPhone 拍照時自動框出人臉 |
| `人臉辨識（Face Recognition）` | 辨認這張臉是誰 | 手機 Face ID 解鎖 |
| `瑕疵檢測（Defect Detection）` | 在產品影像中找出不良品 | 半導體晶圓瑕疵檢測 |
| `車牌辨識（License Plate Recognition）` | 從影像中讀出車牌號碼 | 停車場自動辨識收費 |
| `醫療影像判讀（Medical Imaging）` | 從 X 光/CT 中找出病灶 | 台大醫院 AI 胰臟癌偵測系統 🇹🇼 |
| `Deepfake 偵測（Deepfake Detection）` | 辨別影像/影片是否被 AI 偽造 | 社群平台自動標記可疑換臉影片 |

🗣️ **白話說明：** 想像你在 Instagram 滑限時動態，看到一張照片——鑑別式AI 就像你的眼睛，負責「看出裡面有什麼」。它不會幫你修圖，但它會告訴你：「這張照片裡有一隻柴犬、背景是大安森林公園。」

#### 生成式CV應用——創造、修復、美化

| 應用 | 說明 | 生活例子 |
|------|------|----------|
| `影像生成（Image Generation）` | 從文字描述或隨機雜訊產生全新圖片 | Midjourney、DALL-E、Stable Diffusion 生成插畫 |
| `風格轉換（Style Transfer）` | 把一張照片轉成另一種畫風 | 把自拍變成油畫風 |
| `超解析度（Super Resolution）` | 把低解析度圖片放大變清晰 | 舊照片修復、監視器畫面增強 |
| `影像修復（Image Inpainting）` | 填補圖片中缺失或被遮擋的部分 | Photoshop「生成式填滿」功能 |
| `合成數據（Synthetic Data）` | 產生訓練用的假圖片來擴充資料集 | 用 GAN 生成罕見瑕疵圖片 |

🗣️ **白話說明：** 生成式CV 就像 IG 的美顏濾鏡 on steroids。不只修個皮膚、加個光——它能直接「無中生有」畫出一張從沒存在過的照片。你跟 Midjourney 說「一隻穿西裝的柴犬站在台北 101 前面」，它就真的畫出來了。

#### 🔗 鑑別+生成 整合應用（CV）🔥🔥

這才是 L11402 的重點——兩者怎麼搭配。

**應用一：自駕車（Autonomous Driving）**

```
┌─────────────────────────────────────────────────────┐
│              自駕車 AI 系統                            │
│                                                      │
│  🔍 鑑別式                    🎨 生成式               │
│  ┌──────────────┐            ┌──────────────┐       │
│  │ 物件偵測      │            │ 場景模擬生成   │       │
│  │ 行人/車輛/號誌 │            │ 合成訓練數據   │       │
│  │ 車道線辨識    │            │ 極端情境模擬   │       │
│  └──────┬───────┘            └──────┬───────┘       │
│         │                           │                │
│         └───────┐    ┌──────────────┘                │
│                 ▼    ▼                               │
│          ┌─────────────────┐                         │
│          │ 更安全的自駕決策  │                         │
│          └─────────────────┘                         │
└─────────────────────────────────────────────────────┘
```

- **鑑別式**負責即時偵測路上的行人、車輛、紅綠燈
- **生成式**負責模擬各種罕見危險場景（暴雨、逆光、行人突然衝出）來訓練鑑別模型
- NVIDIA Omniverse 就是用生成式 AI 模擬城市場景，讓自駕車在虛擬世界中練習

🗣️ **白話說明：** 自駕車不可能真的在馬路上撞 10,000 次來學怎麼閃避行人。所以生成式 AI 模擬出各種危險情境（「如果下雨天有小孩突然跑出來」），鑑別式 AI 在這些虛擬場景中反覆練習辨識。就像你玩賽車遊戲練技術，再上真正的駕訓班。

**應用二：製造業品管（Manufacturing QC）**

```
  工廠產線
     │
     ▼
┌──────────┐     生成罕見瑕疵圖片      ┌──────────┐
│ 鑑別式AI  │ ◀──── (資料增強) ─────── │ 生成式AI  │
│ 瑕疵分類  │                          │ GAN 合成  │
│ 良品/不良品│                          │ 稀有不良品 │
└──────────┘                          └──────────┘
```

- 工廠真實瑕疵品可能一萬片才出一片，資料太少沒辦法訓練
- **生成式 AI（GAN）** 合成大量「假的瑕疵圖片」來擴充訓練資料
- **鑑別式 AI** 用這些擴充資料訓練後，偵測瑕疵的準確度大幅提升

**應用三：Deepfake 攻防**

- **生成式** 製造 Deepfake 換臉影片
- **鑑別式** 偵測影片是否為 Deepfake
- 兩者不斷「攻防」，就像 GAN 的生成器 vs 鑑別器（L11401 提過的概念）

**應用四：台灣醫療案例** 🇹🇼

- **台大醫院 AI 胰臟癌偵測**：鑑別式 AI 分析 CT 影像，早期偵測胰臟腫瘤
- **GAN 生成罕見病例影像**：某些罕見疾病的醫療影像太少，用生成式 AI 合成更多訓練數據，提升鑑別模型的診斷能力
- **遠傳+亞東醫院 GenAI 照護**：生成式 AI 協助撰寫醫療紀錄和護理摘要 🇹🇼

### 3.3 語音與音訊（Speech & Audio）🔥🔥

`語音辨識（Automatic Speech Recognition, ASR）` 和 `語音合成（Text-to-Speech, TTS）` 是這個領域裡**最容易搞混**的一對。考試經常在這裡設陷阱。

> **記住這個方向箭頭，永遠不會搞錯：**
>
> - ASR 語音辨識：🎤 聲音 → 📝 文字 （**鑑別式**——辨認你說了什麼）
> - TTS 語音合成：📝 文字 → 🔊 聲音 （**生成式**——創造出語音）

#### 鑑別式語音應用——聽懂、辨認、分析

| 應用 | 說明 | 生活例子 |
|------|------|----------|
| `語音辨識（ASR）` 🔥🔥 | 把你說的話轉成文字 | Siri「嘿 Siri 幫我設鬧鐘」、Google 語音搜尋 |
| `聲紋辨識（Speaker Recognition）` | 透過聲音特徵判斷「是誰在說話」 | 銀行電話客服驗證身份 |
| `語音情緒分析（Speech Emotion Analysis）` | 從語音的語調、節奏判斷情緒 | 客服品質監控——判斷客戶是否生氣 |
| `音訊分類（Audio Classification）` | 辨識聲音屬於哪種類型 | 辨別嬰兒哭聲、車禍聲、玻璃破裂聲 |

🗣️ **白話說明：** 你對著手機說「嘿 Siri，附近有什麼好吃的？」Siri 先用 ASR（鑑別式）把你的聲音轉成文字，理解你在問美食推薦。這個「聽懂你說什麼」的步驟就是鑑別式——它在「辨認」你的語音內容。

#### 生成式語音應用——合成、複製、創作

| 應用 | 說明 | 生活例子 |
|------|------|----------|
| `語音合成（TTS）` 🔥🔥 | 把文字轉成自然的語音 | Google 地圖導航語音、有聲書自動朗讀 |
| `聲音複製（Voice Cloning）` | 用少量語音樣本複製出某人的聲音 | AI 語音助手用你喜歡的聲音說話 |
| `音樂生成（Music Generation）` | AI 創作全新的音樂 | Suno AI 生成完整歌曲 |
| `語音翻譯（Speech Translation）` | 即時將一種語言的語音翻譯成另一種 | 出國旅遊的即時翻譯機 |

🗣️ **白話說明：** Google 地圖跟你說「前方 300 公尺左轉」——這段語音不是某個人錄的，是 TTS（生成式）即時「製造」出來的聲音。它在「創造」一段語音，所以是生成式。

#### 🔗 鑑別+生成 整合應用（語音）🔥

**應用一：智慧客服管線（Smart Customer Service Pipeline）**

```
  客戶打電話進來
       │
       ▼
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│  ASR 語音辨識 │ ──▶ │  NLP 意圖理解 │ ──▶ │  TTS 語音合成 │
│  (鑑別式)     │     │ (鑑別→生成)   │     │  (生成式)     │
│ 🎤→📝        │     │ 📝→💡→📝    │     │ 📝→🔊        │
│ 聲音轉文字    │     │ 意圖分類(鑑別)│     │ 文字轉語音    │
│              │     │ 回覆生成(生成)│     │ 唸出回覆      │
└──────────────┘     └──────────────┘     └──────────────┘
```

這是最經典的**管線串接（Pipeline）模式**：ASR（鑑別）→ NLP（鑑別+生成）→ TTS（生成）。三個 AI 模組像接力賽一樣串在一起。

🗣️ **白話說明：** 你打 LINE 客服的語音通話。AI 先「聽懂」你說的話（ASR，鑑別式），再「理解」你想問什麼、生成回答文字（NLP），最後「唸出來」給你聽（TTS，生成式）。三段接力，一氣呵成。

> 💡 **趨勢補充：** 最新的多模態AI（如 GPT-4o）正逐漸打破傳統管線界限，能直接進行語音到語音的端到端處理。但考試仍以傳統管線架構為主要考點。

**應用二：語音防詐（Voice Anti-Fraud）**

- `聲紋辨識（Speaker Recognition）`（鑑別式）驗證來電者身份
- `Deepfake 語音偵測`（鑑別式）檢查語音是否為 AI 合成偽造
- 雙管齊下防止 AI 聲音複製詐騙——「假裝你兒子打電話要你匯款」

🗣️ **白話說明：** 現在詐騙集團可以用 AI 複製你家人的聲音。銀行的防詐系統同時用兩種鑑別式 AI：一個確認「這真的是本人的聲紋嗎？」另一個確認「這段語音有沒有被 AI 合成的痕跡？」兩關都過了才放行。

**台灣案例：達宣醫聲通** 🇹🇼

達宣科技的「醫聲通」系統整合 ASR 語音辨識，讓醫師口述病歷就能自動轉成結構化電子病歷，是台灣醫療語音 AI 的代表案例。

### 3.4 文字與自然語言處理（Text & NLP）🔥🔥

`自然語言處理（Natural Language Processing, NLP）` 是讓電腦「讀懂」和「寫出」人類語言的技術。ChatGPT 爆紅後，這是大家最熟悉的 AI 領域，但考試會考的不只是「ChatGPT 能做什麼」——鑑別式的 NLP 應用同樣重要。

#### 鑑別式NLP應用——分類、過濾、辨識

| 應用 | 說明 | 生活例子 |
|------|------|----------|
| `情感分析（Sentiment Analysis）` 🔥 | 判斷文字的情緒傾向（正面/負面/中性） | 蝦皮商品評論自動分析「好評率」 |
| `垃圾郵件過濾（Spam Filtering）` | 判斷信件是不是垃圾郵件 | Gmail 自動把廣告信丟垃圾桶 |
| `文字分類（Text Classification）` | 把文字歸到預定義的類別 | 客服系統自動分類「退貨」「查訂單」「投訴」 |
| `命名實體辨識（Named Entity Recognition, NER）` | 從文字中找出人名、地名、組織名等 | 新聞自動標記「台積電」「張忠謀」 |
| `AI 文字偵測（AI Text Detection）` | 判斷文字是人寫的還是 AI 生成的 | 大學教授檢查學生報告是否 AI 代寫 |

🗣️ **白話說明：** 你在蝦皮買東西，看到一堆評論。情感分析（鑑別式）就像一個超快的助理，幫你把幾千則評論自動分成「好評」和「負評」，你直接看統計就知道這家可不可靠。它不會寫評論，只是「分辨」每則評論的立場。

> 💡 **實務補充：** 現代大型語言模型（如 ChatGPT）也能透過提示詞執行分類等鑑別式任務，但在企業管線中，成本與速度考量下仍常使用專門的鑑別式模型。

#### 生成式NLP應用——撰寫、摘要、翻譯、編碼

| 應用 | 說明 | 生活例子 |
|------|------|----------|
| `文字生成（Text Generation）` 🔥 | 根據提示產生全新文字 | ChatGPT、Gemini 對話回覆 |
| `文章摘要（Summarization）` | 把長文濃縮成重點摘要 | AI 幫你把 10 頁論文摘成 3 段 |
| `機器翻譯（Machine Translation）` | 一種語言翻譯成另一種 | Google 翻譯、DeepL |
| `程式碼生成（Code Generation）` | 根據描述產生程式碼 | GitHub Copilot、Cursor |

🗣️ **白話說明：** ChatGPT 就是生成式 NLP 的代表。你問它「幫我寫一封英文求職信」，它不是從資料庫裡複製一封給你——它是「創造」出一封全新的信。這就是「生成」。

#### 🔗 鑑別+生成 整合應用（NLP）🔥🔥

**應用一：RAG 檢索增強生成（Retrieval-Augmented Generation）** 🔥🔥

```
  使用者提問：「公司差旅報帳上限是多少？」
       │
       ▼
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│  1. 檢索      │ ──▶ │  2. 擷取      │ ──▶ │  3. 生成      │
│  (鑑別式)     │     │  (鑑別式)     │     │  (生成式)     │
│ 搜尋公司知識庫 │     │ 找出最相關段落 │     │ 根據找到的資料 │
│ 比對相關文件   │     │ 排序相關程度   │     │ 生成自然語言   │
│              │     │              │     │ 回答          │
└──────────────┘     └──────────────┘     └──────────────┘
```

`RAG（Retrieval-Augmented Generation，檢索增強生成）` 是目前企業 AI 應用中最熱門的整合模式之一。

- **前半段是鑑別式**：在知識庫中「檢索」出跟問題最相關的文件段落（這是分類/比對的工作）
- **後半段是生成式**：根據檢索到的資料「生成」自然語言的回答

> ⚠️ **考試陷阱**：RAG 不是純生成技術！它的「R（Retrieval）」是鑑別式的檢索環節。考試如果問「RAG 屬於哪種 AI？」正確答案是「整合了鑑別式（檢索）和生成式（回答）」。

🗣️ **白話說明：** 想像你進一家很大的圖書館問：「台灣最高的山是哪座？」圖書館員先去書架上「找到」相關的地理書（鑑別式——搜尋比對），翻到正確頁面後，用自己的話「說給你聯聽」（生成式——用找到的資訊產生自然語言回答）。RAG 就是這樣：先找、再說。（此為初級概念說明，RAG 的技術實作細節如向量資料庫、嵌入方式等屬中級內容，初級只需了解概念）

**應用二：內容創作+自動審核**

```
  ┌──────────┐          ┌──────────┐
  │ 生成式AI  │ ──生成──▶ │ 鑑別式AI  │
  │ 撰寫文案  │          │ 自動審核  │
  │ 草稿產出  │  ◀──退回── │ 合規/品質 │
  └──────────┘  不合格    └──────────┘
                              │ 合格
                              ▼
                         ✅ 發布上線
```

- 生成式 AI 撰寫行銷文案或新聞摘要
- 鑑別式 AI 自動審查內容是否合規（有沒有不當言論、是否符合品牌風格）
- 不合格就退回重寫，合格才發布

**應用三：客服自動化（Customer Service Automation）**

- 鑑別式 AI：`意圖分類（Intent Classification）`——判斷客戶是要「查訂單」「申請退貨」還是「投訴」
- 生成式 AI：根據判斷出的意圖，`生成個人化回覆`
- 整合效果：客服機器人既能「聽懂你要幹嘛」又能「給你像人一樣的回答」

🗣️ **白話說明：** 你在 Uber Eats App 點「聯繫客服」，打字說「我的餐沒送到」。AI 先用鑑別式判斷你的意圖是「外送問題」，再用生成式產出一段個人化回覆：「很抱歉您的訂單 #A1234 尚未送達，我們已通知外送員，預計 10 分鐘內到達。」——不是罐頭回覆，是根據你的訂單資訊即時生成的。

### 3.5 五大整合模式（Five Integration Patterns）🔥🔥

學完三大領域的具體應用後，讓我們拉高一層——這些整合方式可以歸納成**五大模式**。考試喜歡考你「辨認這是哪種整合模式」。

#### 模式一：「生—鑑—監」閉環（Generate-Discriminate-Monitor Loop）

```
      ┌─────────┐
      │ 1. 生成  │ ── AI 產出內容（文字/圖像/決策）
      └────┬────┘
           ▼
      ┌─────────┐
      │ 2. 鑑別  │ ── AI 自動審核品質/合規性
      └────┬────┘
           ▼
      ┌─────────┐
      │ 3. 監控  │ ── 持續追蹤效果，異常回饋給生成端
      └────┬────┘
           │
           └──── 回饋循環 ────▶ 回到 1. 持續改善
```

**概念：** 生成式 AI 先創造，鑑別式 AI 再審核，上線後持續監控，形成一個不斷改善的閉環。

🗣️ **白話說明：** 就像一套內容平台系統——AI 自動生成行銷文案或商品描述（生成），另一個 AI 審核文案有沒有違規或誇大內容（鑑別），上線後持續追蹤點擊率和使用者滿意度（監控），數據回饋讓文案品質越來越高。

#### 模式二：GAN 協作模式（Generator-Discriminator Collaboration）🔥🔥

```
  ┌──────────┐          ┌──────────┐
  │ 生成器    │ ──假圖──▶ │ 鑑別器    │
  │ Generator │          │Discriminator│
  │ (生成式)  │  ◀──回饋── │ (鑑別式)  │
  └──────────┘  真/假判定  └──────────┘
       │
       ▼  （經過反覆對抗後）
  生成品質越來越逼真
```

`GAN（Generative Adversarial Network，生成對抗網路）` 是鑑別式與生成式 AI 整合的**最經典原型**。

- **生成器（Generator）**：生成式 AI，負責產生假圖片
- **鑑別器（Discriminator）**：鑑別式 AI，負責判斷真假
- 生成器負責創造內容，鑑別器負責判斷真偽——兩者各司其職，互相搭配，最終生成的圖片幾可亂真

> ⚠️ **超級考試重點**：GAN 不是「純生成式」！它內部同時包含生成器（生成式）和鑑別器（鑑別式），是最完美的「鑑別+生成整合」範例。（GAN 的訓練機制如損失函數、Nash 均衡等屬中級內容，初級只需了解這個對抗協作的概念）

🗣️ **白話說明：** GAN 就像一個假鈔偽造者（生成器）和一個驗鈔員（鑑別器）在互相較勁。偽造者越做越像真鈔，驗鈔員越來越會抓假鈔。最後偽造者厲害到連專家都分不出來。這就是為什麼 GAN 生成的圖片可以這麼逼真。

#### 模式三：合成數據增強（Synthetic Data Augmentation）🔥

```
  問題：訓練資料太少（尤其是罕見類別）
       │
       ▼
  生成式 AI（GAN 等生成模型）合成大量假資料
       │
       ▼
  合成資料 + 真實資料 合併
       │
       ▼
  鑑別式 AI 用更多資料訓練 → 效果大幅提升
```

**概念：** 當真實資料不夠（尤其是稀有案例），用生成式 AI 合成訓練數據，幫助鑑別式 AI 學得更好。

- 製造業：生成罕見瑕疵圖片（一萬片才出一片的不良品）
- 醫療：生成稀有疾病影像（某些罕見癌症的 CT 影像全世界才幾百張）
- 金融：美國運通 / SAS 用 GAN 生成詐欺交易樣本來訓練防詐模型

🗣️ **白話說明：** 你要訓練一隻 AI 辨認一種超罕見的昆蟲，但全世界只有 50 張照片。用生成式 AI「畫」出 5,000 張這種昆蟲的合成照片，混在一起訓練，鑑別式 AI 就能學得更好。就像老師出考卷的時候，除了用歷屆考古題，還自己另外出了一些「類似但不同」的新題目讓你練習。

#### 模式四：管線串接（Pipeline Integration）🔥

```
  輸入 ──▶ [模組A] ──▶ [模組B] ──▶ [模組C] ──▶ 輸出
           (鑑別式)    (鑑別→生成)   (生成式)
```

**概念：** 多個 AI 模組像工廠流水線一樣串在一起，前一個的輸出是後一個的輸入。

最經典的例子就是前面提到的智慧客服：ASR（鑑別）→ NLP（鑑別+生成）→ TTS（生成）

其他例子：
- `即時翻譯（Real-time Translation）`：ASR 辨識語音 → 翻譯模型轉換語言 → TTS 合成目標語言語音
- 自動影片字幕：ASR 辨識對話 → NLP 校正語法 → 輸出字幕文字

#### 模式五：RAG 檢索增強生成（Retrieval-Augmented Generation）🔥🔥

前面 3.4 已經詳細介紹過，這裡總結核心概念：

- **R = Retrieval（檢索）**——鑑別式：從知識庫中找出相關資料
- **A = Augmented（增強）**——把找到的資料當作「提示」給生成模型
- **G = Generation（生成）**——生成式：根據檢索結果產生回答

**為什麼需要 RAG？** 因為 ChatGPT 這類生成式 AI 有「知識截止日」和「幻覺（Hallucination）」的問題——它可能回答過時的資訊或直接編造。RAG 讓它先查正確資料再回答，大幅降低錯誤率。

🗣️ **白話說明：** ChatGPT 就像一個很聰明但消息不一定最新的朋友。你問它「我們公司今年的報帳新規定」，它可能瞎掰。RAG 就是先讓它去翻公司內部文件（檢索），找到正確資料後再回答你（生成）。有了 RAG，AI 就從「憑記憶亂說」變成「查資料再說」。

### 3.6 產業應用總覽 🔥

以下整理六大產業如何整合鑑別式與生成式 AI。考試常出「給一個產業場景，問這是鑑別式、生成式、還是整合應用」。

| 產業 | 鑑別式應用 | 生成式應用 | 整合方式 |
|------|-----------|-----------|---------|
| **醫療** | AI 診斷（台大胰臟癌偵測）、醫療影像判讀 | GenAI 照護紀錄（遠傳+亞東 🇹🇼）、生成罕見病例影像 | GAN 合成稀有病例 → 訓練診斷 AI |
| **金融** | 信用評分、詐欺偵測、異常交易預警 | 財務報告自動撰寫、客戶個人化建議 | GAN 生成詐欺樣本 → 訓練防詐模型（美國運通/SAS） |
| **製造** | 瑕疵檢測、良率分析、預測性維護 | 產品設計生成、合成數據 | GAN 生成罕見瑕疵 → 強化品質檢測 AI |
| **自駕車** | 物件偵測、車道辨識、號誌辨識 | 合成駕駛場景模擬 | 生成式模擬場景 → 訓練偵測模型（NVIDIA Omniverse） |
| **教育** | 學習行為分析、學生表現預測 | 個人化教材生成、AI 自動出題 | 分析弱點（鑑別）→ 生成針對性練習題（生成） |
| **娛樂** | 推薦系統（Netflix/Spotify）、內容審核 | 遊戲場景生成、AI 音樂、AI 劇本 | 推薦+生成：分析偏好 → 生成個人化內容 |

🗣️ **白話說明：** 不管哪個產業，整合的邏輯都一樣——**鑑別式負責「看」和「判斷」，生成式負責「做」和「創造」**。一個是眼睛和大腦，一個是手和嘴巴。真正厲害的 AI 系統，通常眼睛和手都要有。

## 5. 口訣與記憶法（Mnemonics）

### 口訣一：三大領域速記——「看聽讀」

> **「看（CV）、聽（語音）、讀（NLP）——鑑別分辨，生成創作」**

看到圖片相關 → 電腦視覺領域
聽到聲音相關 → 語音音訊領域
讀到文字相關 → NLP 文字領域

每個領域都有鑑別（分辨）和生成（創作）兩面。

### 口訣二：ASR vs TTS 方向——「嘴到手，手到嘴」

> **「ASR：嘴→手（說話轉文字），TTS：手→嘴（文字轉說話）」**

ASR = 你用嘴說，AI 用手打字（語音→文字，鑑別式）
TTS = AI 讀你手打的字，用嘴唸出來（文字→語音，生成式）

也可以這樣記：ASR = LINE 語音訊息轉文字功能；TTS = Google 地圖導航語音

### 口訣三：五大整合模式——「生鑑監、GAN 抗、合增強、管線串、RAG 查」

> **「生鑑監、GAN 抗、合增強、管串接、RAG 查」**

- 生鑑監 = 生—鑑—監 閉環
- GAN 抗 = GAN 對抗協作
- 合增強 = 合成數據增強
- 管串接 = 管線串接
- RAG 查 = RAG 檢索增強生成

### 口訣四：GAN 的本質——「一個造假、一個驗真、整合之王」

> **GAN = 造假的生成器 + 驗真的鑑別器 = 鑑別+生成的完美整合**

### 口訣五：產業應用速記——「鑑別看門狗，生成創作家」

> **「鑑別是看門狗（偵測、審核、預警），生成是創作家（撰寫、設計、模擬）」**

在任何產業裡：
- 需要「抓壞人、找問題」→ 鑑別式
- 需要「寫東西、畫東西」→ 生成式
- 需要「兩者配合」→ 整合應用

## 6. 考試陷阱（Exam Traps）

### 陷阱一：電腦視覺只有生成式？

❌ **錯誤觀念：** 電腦視覺就是 AI 生成圖片（Midjourney、DALL-E），所以 CV 是生成式的領域。

✅ **正確理解：** CV 領域裡鑑別式應用更早、更廣泛（物件偵測、人臉辨識、瑕疵檢測、車牌辨識、醫療影像判讀等等）。Midjourney 只是 CV 的「生成面」，不代表整個 CV 領域。

### 陷阱二：GAN 是純生成式？

❌ **錯誤觀念：** GAN 的全名是「生成對抗網路」，裡面有「生成」兩個字，所以它是純生成式 AI。

✅ **正確理解：** GAN 內部同時包含 **生成器（Generator，生成式）** 和 **鑑別器（Discriminator，鑑別式）**。它是鑑別式+生成式的經典整合範例。考試如果問「以下哪個同時運用了鑑別式和生成式？」答案就是 GAN。

### 陷阱三：RAG 是純生成技術？

❌ **錯誤觀念：** RAG 的全名有「Generation」，所以它是純生成式技術。

✅ **正確理解：** RAG = **Retrieval（檢索，鑑別式）** + Augmented + **Generation（生成，生成式）**。前半段的檢索比對是鑑別式的工作，後半段的生成回答才是生成式。RAG 是整合應用的代表。

### 陷阱四：語音辨識和語音合成是同一件事？

❌ **錯誤觀念：** 語音辨識（ASR）和語音合成（TTS）都跟語音有關，差不多是同一種技術。

✅ **正確理解：** 方向完全相反！ASR 是 🎤→📝（聲音轉文字，鑑別式），TTS 是 📝→🔊（文字轉聲音，生成式）。一個是「聽懂你說什麼」，一個是「說出讓你聽的話」。

### 陷阱五：整合應用一定比單獨使用好？

❌ **錯誤觀念：** 鑑別+生成整合一定比單獨使用效果好，所以永遠該選整合方案。

✅ **正確理解：** 要看場景。如果任務只需要「分類判斷」（例如垃圾郵件過濾），用鑑別式就夠了，不需要加生成式。整合有其成本和複雜度，不是萬能解。考試如果給你一個簡單的分類場景，不要硬選「整合應用」。

### 陷阱六：合成數據可以完全取代真實數據？

❌ **錯誤觀念：** 用生成式 AI 合成的訓練數據可以完全取代真實數據。

✅ **正確理解：** 合成數據是用來**增強**（Augment）而非**取代**（Replace）真實數據。合成數據可能帶有生成模型的偏差，必須與真實數據混合使用才能確保品質。

## 7. 情境題快速判斷（Scenario Quick-Judge）

### 7.1 關鍵字 → AI 類型速查表

| 看到這些關鍵字 | 判斷為 | 原因 |
|--------------|--------|------|
| 辨識、偵測、分類、判斷、篩選、預測、過濾、分辨 | **鑑別式** | 都是「分辨」類動作 |
| 生成、創作、合成、撰寫、模擬、產生、翻譯、摘要 | **生成式** | 都是「創造」類動作（⚠️ 翻譯在管線情境如語音翻譯中屬整合應用） |
| 先檢索再回答、先分析再生成、GAN、攻防、閉環 | **整合應用** | 兩者搭配 |
| 語音辨識、ASR、聽懂、聲紋 | **鑑別式（語音）** | 聲音→辨認 |
| 語音合成、TTS、唸出來、AI 配音 | **生成式（語音）** | 文字→聲音 |
| 瑕疵檢測、良品判定、異常偵測 | **鑑別式（CV）** | 看圖→判斷 |
| AI 繪圖、影像生成、風格轉換 | **生成式（CV）** | 無中生有 |
| 情感分析、垃圾郵件、分流、NER | **鑑別式（NLP）** | 文字→分類 |
| ChatGPT、文字生成、AI 寫作 | **生成式（NLP）** | 產出新文字 |

### 7.2 典型情境題速解模板

**情境A：「工廠用 AI 檢查產品有沒有刮傷」**
→ 關鍵字：檢查、刮傷（瑕疵偵測）→ ✅ 鑑別式（CV）

**情境B：「AI 根據使用者描述生成產品設計圖」**
→ 關鍵字：生成、設計圖 → ✅ 生成式（CV）

**情境C：「客服系統先聽懂客戶說什麼，再自動語音回覆」**
→ 關鍵字：聽懂（ASR）+ 語音回覆（TTS）→ ✅ 整合應用（管線串接）

**情境D：「企業 AI 先查內部文件，再回答員工問題」**
→ 關鍵字：先查（檢索）再回答（生成）→ ✅ 整合應用（RAG）

**情境E：「用 AI 生成大量罕見不良品圖片來訓練品檢系統」**
→ 關鍵字：生成圖片 + 訓練品檢 → ✅ 整合應用（合成數據增強）

**情境F：「判斷一段影片是不是 Deepfake」**
→ 關鍵字：判斷、Deepfake 偵測 → ✅ 鑑別式（CV）

**情境G：「GAN 的鑑別器負責什麼工作？」**
→ GAN 鑑別器 = 判斷真假 → ✅ 鑑別式（但 GAN 整體是整合應用）

**情境H：「銀行用 AI 分析語音判斷來電者是否為本人」**
→ 關鍵字：分析語音、判斷身份（聲紋辨識）→ ✅ 鑑別式（語音）

### 7.3 整合 vs 單獨判斷法則

```
  題目描述的任務只有一個動作？
       │
       ├── 是 → 判斷動作類型
       │        ├── 分辨/偵測/分類 → 鑑別式
       │        └── 創造/生成/合成 → 生成式
       │
       └── 否（有兩個以上 AI 動作串聯）→ 整合應用
                ├── 先查再答 → RAG
                ├── 一個造假一個驗真 → GAN 協作
                ├── 生成訓練資料給另一個 AI → 合成數據增強
                ├── 多個模組串接 → 管線串接
                └── 生成→審核→監控 → 生—鑑—監閉環
```
