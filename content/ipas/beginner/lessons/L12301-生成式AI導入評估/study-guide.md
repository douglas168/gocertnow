## 3. 核心概念

### 3.1 經濟部產業發展署《AI導入指引》框架 🔥🔥

經濟部產業發展署（Industrial Development Administration, MOEA）發布的《製造業AI導入指引》（AI Adoption Guide）是考試最核心的框架。雖然標題寫「製造業」，但其四階段流程是通用型框架，iPAS 考試明確引用此文件。

🗣️ 白話說明：想像你開了一家手搖飲店，想導入 AI 點餐系統。你不會第一天就花 50 萬買設備吧？一定是先想清楚「我到底需不需要？」→ 設計方案 → 小規模測試 → 確定可行才全面上線。《AI導入指引》就是這個常識的系統化版本。

#### 四階段流程 🔥🔥

```
┌──────────────┐    ┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│  1. 構想評估  │ →  │  2. 方案設計  │ →  │  3. 驗證/POC │ →  │  4. 實施/營運 │
│              │    │              │    │              │    │              │
│ · 痛點分析    │    │ · 定義 KPI   │    │ · 原型開發    │    │ · 正式部署    │
│ · 資源盤點    │    │ · 資料需求    │    │ · 模擬測試    │    │ · 持續監控    │
│ · 場景選定    │    │ · 成本估算    │    │ · ROI 驗證    │    │ · 文化推廣    │
└──────────────┘    └──────────────┘    └──────────────┘    └──────────────┘
                                                                    │
                    ← ← ← 可回到前一階段調整 ← ← ← ← ← ← ← ← ← ←┘
```

各階段重點：

| 階段 | 核心問題 | 關鍵活動 | 產出 |
|------|---------|---------|------|
| 構想評估 | 我們需要 AI 嗎？ | 痛點分析、資源盤點、AI 適配度檢查 | 應用場景清單 |
| 方案設計 | 怎麼做？用什麼？ | KPI 定義、資料需求分析、成本初估 | 專案計畫書 |
| 驗證/POC | 真的可行嗎？ | 原型開發、模擬營運、ROI 驗證 | POC 報告 |
| 實施/營運 | 如何規模化？ | 部署上線、監控再訓練、擴展 AI 文化 | 營運中的 AI 系統 |

🗣️ 白話說明：這就像大學分組報告的流程——先討論主題（構想）→ 分工規劃（方案設計）→ 先做一版 demo 給教授看（POC）→ 確認 OK 後完成正式報告（實施）。差別是企業的分組報告要花真金白銀。

### 3.2 四大評估維度 🔥🔥

導入AI前需要從四個角度做評估，每個角度問的問題不同：

#### 技術評估（Technical Assessment）🔥🔥

**定義：** 檢視組織現有的技術基礎設施（Infrastructure）、資料品質（Data Quality）、與團隊技術能力（Technical Capability），判斷是否有能力支撐AI導入。

🗣️ 白話說明：就像你想在租屋處裝一台工業級咖啡機——先確認電壓夠不夠（基礎設施）、有沒有好的咖啡豆來源（資料品質）、你會不會操作（團隊能力）。三缺一都不行。

技術評估檢查清單（初級概念層級）：

```
✅ 基礎設施：網路頻寬、伺服器算力、雲端服務是否足夠？
✅ 資料品質：資料夠不夠多？格式統一嗎？有沒有標註？
✅ 團隊能力：有沒有人會用？需要多少教育訓練？
✅ 安全合規：資料存放符不符合法規要求？
```

#### 工具評估（Tool Evaluation）🔥

**定義：** 比較市面上不同的AI工具或平台，從功能、整合能力、供應商穩定度等面向，選出最適合組織需求的工具。

🗣️ 白話說明：就像你要選一個專案管理工具——Notion、Trello、Asana 都能用，但你的團隊5個人都用 Google 生態系，那選 Google 有整合的工具可能更適合。工具評估就是「貨比三家」。

工具評估考量面向：

| 面向 | 問什麼 | 舉例 |
|------|--------|------|
| 功能完整度 | 能不能做到我要的？ | ChatGPT 能生成文案但不能直接做圖表分析 |
| 整合能力 | 能不能跟現有系統串接？ | Copilot 可以直接在 Word/Excel 裡用 |
| 供應商穩定度 | 這家公司會不會倒？ | 選 OpenAI、Google、Microsoft 相對穩定 |
| 資安合規 | 資料送出去安不安全？ | 企業版 vs 個人版的資料處理政策不同 |
| 成本結構 | 怎麼收費？會不會爆預算？ | 按 Token 計費 vs 月費吃到飽 |

#### 模型評估（Model Evaluation）🔥

**定義：** 針對特定任務，比較不同AI模型（Model）在準確度（Accuracy）、回應速度（Latency）、與成本（Cost）之間的表現，選出最適合的模型。

🗣️ 白話說明：就像你叫 Uber Eats 選餐廳——A 餐廳超好吃但要等 40 分鐘又貴（高準確但高延遲高成本）、B 餐廳普通但 15 分鐘到而且便宜（中等準確但低延遲低成本）。看你是請客吃飯還是趕著吃午餐，選擇就不同。

模型評估的三角取捨：

```
        準確度（Accuracy）
           ▲
          / \
         /   \
        /     \
       /  選擇  \
      /  最適配  \
     /     點     \
    ▼───────────────▼
延遲（Latency）  成本（Cost）
```

**初級只需理解「三者不可能同時最優」的概念。**（此為中級內容，初級只需了解概念）

#### 效能評估（Performance Evaluation）🔥

**定義：** 在AI系統導入後，透過KPI（Key Performance Indicator，關鍵績效指標）比較導入前後的差異，確認AI是否真正帶來價值。

🗣️ 白話說明：你在蝦皮開店後裝了AI客服機器人，但怎麼知道有沒有用？看數據啊——回覆速度從 2 小時變 30 秒、客訴量下降 40%、客服人力從 3 人變 1 人。這就是效能評估。

**重點：效能評估是在「導入後」做的，不是導入前！** 導入前做的是技術評估、工具評估、模型評估。

### 3.3 適用解決方案選擇（Solution Selection）🔥

在完成評估之後，要做出「怎麼做」的決策。初級考試常考三種路線的比較：

| 路線 | 說明 | 適合情境 | 舉例 |
|------|------|---------|------|
| 自建（Build） | 自己組團隊從頭開發 | 核心競爭力、高度客製需求 | 台積電自建良率預測AI |
| 購買（Buy） | 買現成的AI解決方案 | 標準化需求、快速上線 | 購買 Salesforce Einstein |
| 訂閱SaaS（Subscribe） | 訂閱雲端AI服務 | 預算有限、想先試水溫 | 訂閱 ChatGPT Enterprise |

🗣️ 白話說明：就像你要在學校附近吃午餐——自己煮（Build，最客製但最花時間）、去餐廳吃（Buy，直接可用但選擇固定）、還是叫 Uber Eats（SaaS，彈性大但長期可能比較貴）。要看你有多少時間、多少錢、多特別的需求。

**解決方案選擇的關鍵原則：**

1. **不是每個問題都需要 AI** — 如果用 Excel 公式就能解決，不需要導入 AI
2. **從小處開始** — 先用現成的 SaaS 工具驗證價值，再考慮自建
3. **考慮維護成本** — 買或建都需要持續維護，不是一次性投資

### 3.4 成本效益分析（Cost-Benefit Analysis, CBA）🔥🔥

**定義：** 在導入AI前，系統性地比較所有預期成本與預期效益，作為決策依據。初級只需掌握「哪些該考慮」，不需要做數學計算。（此為中級內容，初級只需了解概念）

🗣️ 白話說明：就像你考慮要不要辦健身房會籍——月費 $1,500 很明確（顯性成本），但你還要算交通時間、買運動服的錢、偶爾請教練的費用（隱藏成本）。效益呢？體力變好、看醫生次數減少、精神變好工作效率提升——有些好量化、有些不好量化。CBA 就是把這些通通列出來想清楚。

#### 成本面全景圖 🔥🔥

```
┌─────────────────────────────────────────────────┐
│              AI 導入成本全景                       │
├────────────────────┬────────────────────────────┤
│  顯性成本（少）      │     隱藏成本（多）🔥🔥      │
│   Visible Costs    │     Hidden Costs            │
├────────────────────┼────────────────────────────┤
│ · API / 訂閱費用    │ · 資料收集、清理、標註        │
│ · 雲端運算費用      │ · 員工教育訓練               │
│ · 軟體授權費        │ · 系統整合與客製開發          │
│                    │ · RAG / 知識庫建置           │
│                    │ · 合規與資安審查              │
│                    │ · 組織變革管理               │
│                    │ · 持續維護與模型更新          │
└────────────────────┴────────────────────────────┘
```

🔥🔥 **考試重點：API 費用只佔總成本的 5–15%，真正的大頭是人才培訓、資料治理、系統整合等隱藏成本。** 這是最常考的觀念。

#### 效益面

| 效益類型 | 說明 | 舉例 |
|---------|------|------|
| 效率提升 | 做同樣的事更快 | 客服回覆從 2 小時→30 秒 |
| 品質改善 | 減少錯誤、提高一致性 | 文件翻譯正確率從 80%→95% |
| 成本節省 | 減少人力或資源需求 | 客服團隊從 10 人→3 人+AI |
| 新收入機會 | 創造原本做不到的服務 | AI 個人化推薦帶動營收成長 |

#### ROI 現實檢查（考試常見數據）

- IDC 調查：企業導入生成式AI的平均 ROI = 3.7 倍
- 但 McKinsey 報告：僅 ~6% 企業將 5% 以上稅前利潤歸功於 AI
- **啟示：** 有 ROI 不代表每家都成功，POC 驗證很重要

### 3.5 AI代理（AI Agent）🔥🔥 【114年後期開始出現，115年正式列入考綱】

**定義：** AI代理（AI Agent）是結合大型語言模型（LLM）、記憶（Memory）、工具呼叫（Tool Calling）與自主決策能力的AI系統，能夠自主規劃並執行多步驟任務。

🗣️ 白話說明：一般的 ChatGPT 像是 7-11 的店員——你問一個問題、他回答一個問題，對話結束就忘了你。AI Agent 則像是你的私人助理——你說「幫我安排下週去台南出差」，他會**自己規劃**：查高鐵時刻 → 找飯店 → 看天氣 → 排行程 → 幫你訂位。不需要你一步步下指令。

#### AI Agent 運作迴圈

```
    ┌──────────┐
    │  觀察     │ ← 接收任務/環境資訊
    │ Observe  │
    └────┬─────┘
         │
         ▼
    ┌──────────┐
    │  決策     │ ← LLM 思考下一步
    │ Decide   │
    └────┬─────┘
         │
         ▼
    ┌──────────┐
    │  行動     │ ← 呼叫工具 / 執行任務
    │  Act     │
    └────┬─────┘
         │
         └──→ 回到觀察（直到任務完成）
```

#### AI Agent 四大核心組件

| 組件 | 英文 | 功能 | 白話比喻 |
|------|------|------|---------|
| 大型語言模型 | LLM | 思考與推理的大腦 | 助理的腦袋 |
| 記憶 | Memory | 記住對話脈絡與過去經驗 | 助理的筆記本 |
| 工具呼叫 | Tool Calling | 使用外部工具執行任務 | 助理會用電腦訂票 |
| 自主決策 | Autonomous Decision | 自行規劃下一步行動 | 助理會自己想該做什麼 |

🔥🔥 **考試重點：AI Agent 與一般 GenAI 的核心差異在於「自主性」與「多步驟執行能力」。**

### 3.6 導入評估的整體決策流程

把上面所有概念串起來，形成一個完整的導入決策流程：

```
  公司遇到業務問題
         │
         ▼
  ┌──────────────────┐
  │ 這個問題需要AI嗎？ │ ← 不一定！簡單問題用 Excel 就好
  └────────┬─────────┘
           │ 是
           ▼
  ┌──────────────────┐
  │   技術評估        │ ← 我們有能力做嗎？（基礎設施+資料+團隊）
  └────────┬─────────┘
           │ 可行
           ▼
  ┌──────────────────┐
  │   工具評估        │ ← 市面上有什麼工具可用？
  └────────┬─────────┘
           │
           ▼
  ┌──────────────────┐
  │   模型評估        │ ← 哪個模型最適合這個場景？
  └────────┬─────────┘
           │
           ▼
  ┌──────────────────┐
  │  成本效益分析      │ ← 值得投資嗎？（別忘了隱藏成本）
  └────────┬─────────┘
           │ 值得
           ▼
  ┌──────────────────┐
  │  方案選擇          │ ← Build / Buy / SaaS？
  └────────┬─────────┘
           │
           ▼
```

## 4. 易混淆概念比較表

### 4.1 四大評估類型比較 🔥🔥

| 比較項目 | 技術評估 | 工具評估 | 模型評估 | 效能評估 |
|---------|---------|---------|---------|---------|
| **英文** | Technical Assessment | Tool Evaluation | Model Evaluation | Performance Evaluation |
| **核心問題** | 我們做得到嗎？ | 用哪個工具？ | 用哪個模型？ | AI 真的有用嗎？ |
| **評估時機** | 導入前（最先） | 導入前 | 導入前 | **導入後** |
| **評估對象** | 內部能力 | 外部產品 | AI 模型 | 成效數據 |
| **評估面向** | 基礎設施、資料、團隊 | 功能、整合、供應商 | 準確度、速度、成本 | KPI 變化 |
| **白話比喻** | 廚房設備齊全嗎？ | 要去哪家買食材？ | 選哪種料理方式？ | 客人覺得好吃嗎？ |
| **考試陷阱** | 不只看硬體，人也是 | 最貴 ≠ 最好 | 沒有「最強模型」 | 是導入後才做！ |

### 4.2 AI Agent vs 一般 GenAI 工具 🔥🔥

| 比較項目 | AI代理（AI Agent） | 一般生成式AI工具 |
|---------|-------------------|----------------|
| **互動模式** | 給一個目標，自主完成 | 一問一答 |
| **步驟數** | 多步驟，自行規劃 | 單步驟回應 |
| **記憶** | 有，跨步驟保持上下文 | 僅限當次對話 |
| **工具使用** | 能呼叫外部工具 | 不能（或有限） |
| **自主決策** | 自行判斷下一步 | 被動等待指令 |
| **白話比喻** | 私人助理（自己規劃行程） | 7-11 店員（你問他答） |
| **舉例** | 自動完成「調查+比價+訂購」 | 「幫我寫一封信」→ 給你一封信 |

### 4.3 成本效益分析：顯性成本 vs 隱藏成本 🔥🔥

| 比較項目 | 顯性成本（Visible Costs） | 隱藏成本（Hidden Costs） |
|---------|------------------------|------------------------|
| **佔比** | 約 5–15% | 約 85–95% 🔥🔥 |
| **容易想到？** | 容易 | 不容易（常被低估） |
| **舉例** | API費、訂閱費、雲端費 | 資料治理、教育訓練、整合開發、合規審查 |
| **考試考法** | 不常直接考 | 「以下何者常被忽略？」🔥 |

### 4.4 《AI導入指引》各階段比較

| 比較項目 | 構想評估 | 方案設計 | 驗證/POC | 實施/營運 |
|---------|---------|---------|---------|---------|
| **核心問題** | 要不要做？ | 怎麼做？ | 做得成嗎？ | 怎麼規模化？ |
| **關鍵活動** | 痛點分析、場景選定 | KPI 定義、成本估算 | 原型開發、ROI 驗證 | 部署、監控、文化推廣 |
| **產出** | 場景清單 | 專案計畫 | POC 報告 | 營運中系統 |
| **考法** | 「哪一步應先做？」 | 「KPI 在哪階段定義？」 | 「POC 目的是？」 | 「導入後應做什麼？」 |

## 5. 口訣 / Mnemonics

### 口訣 1：《AI導入指引》四階段 —「構方驗實」

> **構**想 → **方**案 → **驗**證 → **實**施
>
> 🗣️ 記法：「**構方驗實**」念起來像「功防檢視」——導入AI要像打籃球一樣，進攻（構想、方案）和防守（驗證、實施）都要做好。

### 口訣 2：四大評估 —「技工模效」

> **技**術 → **工**具 → **模**型 → **效**能
>
> 🗣️ 記法：「**技工模效**」念起來像「技工模仿」——好的技工（技術+工具）要模仿（模型）出好效果（效能）。
>
> ⚠️ 前三個是導入**前**做，效能評估是導入**後**做！

### 口訣 3：AI Agent 四大組件 —「腦筆手主」

> **腦**（LLM）+ **筆**記本（Memory）+ **手**（Tool Calling）+ **主**見（Autonomous Decision）
>
> 🗣️ 記法：一個好助理要有腦袋、會做筆記、有工具可用、還要有自己的主見。

### 口訣 4：隱藏成本 —「資訓整合維」

> **資**料治理 + 教育**訓**練 + 系統**整合** + 持續**維**護
>
> 🗣️ 記法：「**資訓整維**」——花了資金訓練人，整合好才能維持。API 費用只是冰山一角！

### 口訣 5：解決方案三選一 —「建買租」

> **建**（Build）→ **買**（Buy）→ **租**（SaaS）
>
> 🗣️ 記法：跟房子一樣——自己蓋最客製但最慢、買成屋現成可用、租屋最彈性但長期可能更貴。

## 6. 考試陷阱

### 陷阱 1：四階段順序

❌ 陷阱：可以跳過 POC 直接實施，因為時間緊迫
✅ 正解：四階段是循序漸進的，**不能跳過 POC**。POC 的目的就是在小規模驗證可行性，避免全面導入後才發現不可行而浪費大量資源。考題常以「為了趕進度」為誘餌。

### 陷阱 2：效能評估的時機

❌ 陷阱：效能評估應該在導入前做，跟技術評估一起做
✅ 正解：效能評估是在**導入後**才做的，因為要比較導入前後的 KPI 變化。導入前做的是技術評估、工具評估、模型評估。考題會把四個評估混在一起問順序。

### 陷阱 3：API 費用 = 總成本

❌ 陷阱：導入生成式AI的主要成本是 API 使用費
✅ 正解：API 費用只佔總成本的 5–15%。真正的大頭是**隱藏成本**——資料治理、人才培訓、系統整合、合規審查、持續維護。考題最愛問「以下哪項成本最容易被忽略？」

### 陷阱 4：AI Agent = 聊天機器人

❌ 陷阱：AI Agent 就是比較厲害的聊天機器人
✅ 正解：AI Agent 的核心是**自主決策**和**多步驟執行**。聊天機器人是被動回答問題（你問一句、他答一句），Agent 是給一個目標後**自己規劃步驟、使用工具、持續執行直到完成**。考題會用「自動回覆客戶問題」來誘導你選聊天機器人，但如果題目強調「自主規劃」「多步驟」就要選 Agent。

### 陷阱 5：最強的模型就是最好的選擇

❌ 陷阱：導入AI應該選市場上最強（最準確）的模型
✅ 正解：模型選擇要考慮**準確度、速度、成本的三角取捨**。最強的模型通常最貴、最慢。要根據實際場景需求選「最適合」的，不是「最強」的。例如：內部文件摘要不需要 GPT-4 等級，用更便宜快速的模型就夠了。

### 陷阱 6：所有問題都需要 AI

❌ 陷阱：既然要數位轉型，每個業務流程都應該導入 AI
✅ 正解：不是每個問題都需要 AI 解決。如果用簡單的 Excel 公式或傳統程式就能處理，就不需要花大錢導入 AI。《AI導入指引》的構想評估階段，第一步就是判斷「這個問題是否真的適合用 AI 解決」。

### 陷阱 7：ROI 一定很高

❌ 陷阱：IDC 說生成式AI平均 ROI 3.7 倍，所以導入一定賺
✅ 正解：平均值不代表每家都成功。McKinsey 同期報告顯示僅約 6% 企業將 5% 以上稅前利潤歸功於 AI。這就是為什麼 POC 驗證階段不能省——必須在小規模先證明價值。

## 7. 情境題快速判斷

### 🔑 看到關鍵字 → 選這個答案

**《AI導入指引》階段判斷：**
- 「分析痛點」「盤點資源」「選定場景」 → **構想評估**
- 「定義 KPI」「估算成本」「規劃資料需求」 → **方案設計**
- 「小規模測試」「原型」「POC」「驗證可行性」 → **驗證/POC**
- 「正式上線」「持續監控」「組織文化」「再訓練」 → **實施/營運**

**評估類型判斷：**
- 「基礎設施」「伺服器」「資料品質」「團隊能力」 → **技術評估**
- 「比較工具」「供應商」「功能比較」「API整合」 → **工具評估**
- 「準確度」「延遲」「模型比較」「Token成本」 → **模型評估**
- 「導入後」「KPI變化」「前後比較」「成效追蹤」 → **效能評估**

**成本效益判斷：**
- 「API費用」「訂閱費」「雲端費」 → **顯性成本**
- 「教育訓練」「資料治理」「系統整合」「維護」 → **隱藏成本**
- 「最容易被忽略的成本」 → 選**隱藏成本**類的選項
- 「佔比最高」 → 選**隱藏成本**（85–95%）

**AI Agent 判斷：**
- 「自主規劃」「多步驟」「使用工具」「觀察-決策-行動」 → **AI Agent**
- 「一問一答」「單次回應」「被動回覆」 → **一般 GenAI 工具**
- 「記憶」「跨步驟保持上下文」 → **AI Agent**

**解決方案判斷：**
- 「核心競爭力」「高度客製」「從零開發」 → **Build（自建）**
- 「買現成」「快速上線」「標準化需求」 → **Buy（購買）**
- 「訂閱制」「先試水溫」「預算有限」 → **SaaS（租用）**
- 「不需要AI」「簡單問題」「Excel就能解決」 → **不導入 AI**

**常見情境配對：**

| 情境描述 | 正確答案 |
|---------|---------|
| 某公司想導入AI但不確定有沒有足夠的資料 | 先做**技術評估** |
| 已確認可行，正在比較 ChatGPT Enterprise 和 Gemini Advanced | **工具評估**階段 |
| AI客服上線三個月，想知道是否值得繼續投資 | **效能評估** |
| 主管問「導入AI要花多少錢」，下屬只報了API費用 | 忽略了**隱藏成本** |
| 公司要求AI系統能自動查詢庫存、比價、下單 | 需要 **AI Agent**（多步驟+工具呼叫） |
| 新創公司想快速用AI做客服，預算有限 | 選 **SaaS** 方案 |
| 團隊從未用過AI，想先了解是否適合 | 《AI導入指引》第一階段 **構想評估** |
| 已完成 POC 且成效良好，下一步是？ | 進入 **實施/營運** 階段 |
