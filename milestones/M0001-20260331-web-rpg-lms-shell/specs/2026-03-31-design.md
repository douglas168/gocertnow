# M0001 Design Spec — Web RPG-LMS Shell

**Date:** 2026-03-31
**Status:** Approved
**Milestone:** M0001-20260331-web-rpg-lms-shell

---

## Summary

Replace the existing pre-launch waitlist page with a complete 8-page Next.js shell implementing the LevelCert design system (Indigo + Emerald, dark-mode-only, Poppins + Open Sans + Noto Sans TC). All pages use shadcn/ui components, hardcoded TC placeholder data, and interactive RPG-lite elements (XP, progress bars, streak, badges, radar chart, HP bar). No real auth, payments, or database — visual foundation only. The existing waitlist page and its components are preserved in git history for reuse in a separate GoCertNow.com Vercel project.

---

## Architecture Decisions

### 1. Waitlist Page Disposition

- **Decision:** Preserve the waitlist code. It will be hosted on GoCertNow.com as a separate Vercel project.
- **Action:** Move waitlist-related files (`components/ThemeToggle`, `components/WaitlistForm`, `components/Countdown`, `app/api/waitlist/route.ts`) out of the active codebase. They remain in git history.
- **Dependencies to remove:** `next-themes` (no theme toggle), `resend` (waitlist email API).

### 2. Font Loading — `next/font/google` (Option A)

Three font instances in root `layout.tsx`:

```tsx
import { Poppins, Open_Sans, Noto_Sans_TC } from "next/font/google";

const poppins = Poppins({ subsets: ["latin"], weight: ["600", "700"], variable: "--font-heading" });
const openSans = Open_Sans({ subsets: ["latin"], weight: ["400", "500", "600"], variable: "--font-body" });
const notoSansTC = Noto_Sans_TC({ subsets: ["latin"], weight: ["400", "500", "700"], variable: "--font-cjk" });
```

Body gets all three CSS variable classes. Tailwind config maps:
- `font-heading` → `var(--font-heading), var(--font-cjk), system-ui, sans-serif`
- `font-body` → `var(--font-body), var(--font-cjk), system-ui, sans-serif`

### 3. shadcn/ui Setup

- Run `npx shadcn@latest init` — it detects Tailwind v4 and configures accordingly.
- Dark-mode-only simplifies setup: no theme switching logic.
- If CLI generates v3-style `tailwind.config.ts`, adapt tokens to the v4 `@theme` block in `globals.css`.
- Install components: `button`, `card`, `accordion`, `progress`, `badge`, `separator`, `tabs`, `tooltip`, `skeleton`.

### 4. CSS Variables — Dark Mode Only

Replace all current CSS variables with MASTER.md design system tokens. No `.dark` class — `:root` is the dark theme.


### 5. Route Groups — Three Layouts

```
app/
├── layout.tsx                        # Root: fonts, metadata, dark body
├── globals.css                       # Design system tokens
├── (marketing)/                      # Public pages
│   ├── layout.tsx                    # Marketing navbar + footer
│   ├── page.tsx                      # Landing (/)
│   └── courses/
│       ├── page.tsx                  # Course listing (/courses)
│       └── [slug]/
│           └── page.tsx              # Course detail (/courses/[slug])
├── (app)/                            # Auth-gated pages (simulated logged-in)
│   ├── layout.tsx                    # App navbar (streak badge, nav links) + optional sidebar
│   ├── dashboard/
│   │   └── page.tsx                  # Dashboard (/dashboard)
│   └── learn/
│       └── [slug]/
│           ├── [lesson]/
│           │   └── page.tsx          # Lesson player (/learn/[slug]/[lesson])
│           └── quiz/
│               └── [section]/
│                   └── page.tsx      # Section quiz (/learn/[slug]/quiz/[section])
├── (exam)/                           # Exam pages (minimal chrome)
│   ├── layout.tsx                    # No navbar, deep dark bg (slate-950)
│   └── exam/
│       └── [slug]/
│           ├── page.tsx              # Mock exam (/exam/[slug])
│           └── result/
│               └── page.tsx          # Exam result (/exam/[slug]/result)
```

**Why three route groups:**
- **(marketing):** Full navbar (logo + nav links + "Start Learning" CTA), full footer. `max-w-7xl` for landing, `max-w-6xl` for courses.
- **(app):** App navbar with streak counter, XP display, nav links (Dashboard, My Courses). Lesson player has sidebar nav. No footer.
- **(exam):** Stripped-down layout — no navbar, no footer. Deep dark background (`slate-950`). Maximum focus. Only a small "Exit Exam" link.

### 6. Navigation — Simulated Logged-In State

All nav links always visible. No auth checks. The marketing navbar includes links to `/dashboard`, `/courses`, and auth-gated pages as if the user is logged in. This lets us test all navigation flows end-to-end.

### 7. No `/login` Page

Skipped for M0001. Clerk integration comes in a later milestone.

```
│   ├── streak-calendar.tsx    # 30-day heatmap grid
│   ├── badge-grid.tsx         # Earned (violet) + locked (slate-700 silhouette)
```

### Radar Chart — Pure SVG

No charting library. A `<RadarChart>` React component renders an SVG polygon for 5-6 axes (IPAS exam categories). Props: `categories: { name: string; score: number }[]`. Keeps bundle small.

## Page Specs

### 1. Landing Page (`/`)

**Override:** `design-system/pages/landing.md`
**Max width:** `max-w-7xl`

| Section | Key Elements |
|---------|-------------|
| Hero | H1 headline + subtitle + emerald CTA + gradient bg (slate-900 → slate-950) |
| Problem | 3 pain points with icons |
| Solution | 4 feature cards with RPG accent icons (amber XP, emerald quiz, indigo radar, violet badges) |
| How It Works | 3-step: Learn → Battle → Certify |
| Social Proof | 3-5 testimonial cards + pass rate stat |
| Course Preview | Module outline with lesson counts (uses syllabus-accordion) |
| Pricing | Single tier card, price in TWD, 3-month access, pass guarantee callout, emerald CTA |
| FAQ | 5-7 questions in shadcn Accordion |
| Footer CTA | "Start your quest" repeat CTA |

**Mobile:** Sticky bottom bar with CTA on scroll past hero.

### 2. Course Listing (`/courses`)

**Override:** `design-system/pages/courses.md`
**Max width:** `max-w-6xl`

- Filter bar (placeholder — cert type, level tags)
- Course cards in 2-col grid (desktop), 1-col (mobile)
- Each card: thumbnail area, title, level badge, lesson count, progress bar (for "enrolled" courses), price

### 3. Course Detail (`/courses/[slug]`)

**No separate page override — follows MASTER.md + courses override**
**Max width:** `max-w-6xl`

- **Main column (left):** Course title, description, syllabus accordion (sections → lessons), what you'll learn list, instructor section
- **Sidebar (right, sticky):** Purchase card — price, "Start Learning" CTA (emerald), 3-month access, pass guarantee badge, feature checklist
- **Mobile:** Purchase card becomes fixed bottom bar (price + CTA)

### 4. Lesson Player (`/learn/[slug]/[lesson]`)

**Override:** `design-system/pages/lesson-player.md`

- **Sidebar (left):** Lesson tree nav — sections as groups, lessons as items, active lesson highlighted (indigo), completed lessons with checkmark
- **Content area (right):** Lesson title (H2), body text (TC placeholder), callout boxes (info, warning, tip), key concept highlights
- **Top bar:** Course title, progress bar (amber), current section label
- **Bottom:** Prev/Next lesson navigation buttons
- **RPG:** Subtle "+15 XP" static toast placeholder at bottom of completed lessons
- **Mobile:** Sidebar collapses to hamburger/slide-out drawer

### 5. Section Quiz (`/learn/[slug]/quiz/[section]`)

**Override:** `design-system/pages/section-quiz.md`

- Section title + "Mob Battle" subtitle (subtle, not overwhelming)
- Question cards: question text, 4 answer options as clickable cards
- **Before answer:** Options in neutral state (surface bg, border)
- **After answer:** Correct = emerald border + checkmark, Wrong = red border + X + correct answer highlighted + AI explanation text
- Progress indicator: "Question 2 of 5"
- Completion summary: score, XP earned, badge unlock if applicable
- "Continue to Next Section" or "Retry" buttons
- **No timer, no HP bar** — calm, learning-focused UI

### 6. Dashboard (`/dashboard`)

**Override:** `design-system/pages/dashboard.md`

**Desktop (2-column):**
- **Main (left):** Course cards grid (progress bar, XP, status, access expiry, "Continue Learning" CTA)
- **Sidebar (right):** XP total (large amber number), streak counter + 30-day calendar heatmap, radar chart, badge showcase grid

**Mobile (stacked):**
- RPG stats as horizontal scroll cards (XP, streak, radar thumbnail)
- Course cards 1-col below

**Empty states:**
- No courses: "Your adventure begins here" + CTA to `/courses`
- No streak: "Start your first daily quest to build a streak"
- No badges: All badges shown as locked silhouettes with names

### 7. Mock Exam (`/exam/[slug]`)

**Override:** `design-system/pages/exam.md`

- **Background:** `slate-950` (deeper than normal pages)
- **Top bar:** HP bar (green→red gradient, starts 100%), timer (countdown), question counter (1/60)
- **Center:** Question card on `slate-900` surface, 4 answer options
- **Bottom:** Question grid (numbered circles — answered, current, unanswered states)
- **Interaction:** Clicking an answer marks it and moves to next. HP bar drains on wrong answers (visual only — computed from current wrong answer count).
- "Exit Exam" small link in corner
- **Mobile:** Question grid collapses to horizontal scroll strip

### 8. Exam Result (`/exam/[slug]/result`)

**Override:** `design-system/pages/exam-result.md`

- **Score ring:** Large circular progress indicator with score percentage in center (emerald if pass, red if fail)
- **Category breakdown:** Radar chart showing performance per exam category
- **Wrong answers accordion:** Each wrong question expandable — shows the question, user's answer (red), correct answer (emerald), AI explanation
- **Badge unlocks:** If score triggers a badge (e.g., "Passing Score" at 70%+), show unlock animation (violet glow)
- **CTAs:** "Retry Exam" (outline) + "Review Weak Areas" (emerald) + "Back to Dashboard" (ghost)

## User Flows

### Flow 1: Visitor → Course Detail
1. `/` landing → scans hero, scrolls features
2. Clicks "Browse Courses" in nav or CTA
3. `/courses` → sees 2 IPAS course cards
4. Clicks a card → `/courses/ipas-ai-beginner`
5. Reviews syllabus accordion, sees sticky purchase card
6. Purchase button non-functional (placeholder)

### Flow 2: Lesson → Section Quiz
1. `/dashboard` → clicks "Continue Learning" on a course card
2. `/learn/ipas-ai-beginner/1-1-intro` → reads lesson, sidebar shows section tree
3. Clicks "Next" → progresses through lessons
4. After last lesson in section, clicks "Take Quiz"
5. `/learn/ipas-ai-beginner/quiz/section-1` → answers 5 questions with immediate feedback
6. Sees completion summary with XP earned

### Flow 3: Mock Exam
1. `/dashboard` → clicks "Start Exam"
2. `/exam/ipas-ai-beginner` → deep dark arena, HP bar, timer, questions
3. Answers questions, HP bar drains on wrong answers
4. Finishes → redirected to `/exam/ipas-ai-beginner/result`
5. Sees score ring, radar breakdown, wrong answer accordion, badge unlocks

### Flow 4: Dashboard
1. `/dashboard` → sees course cards with progress, XP total, streak, radar, badges
2. All interactive: cards hover, accordion opens, radar tooltips

## Edge Cases

| Scenario | Handling |
|----------|----------|
| No courses enrolled | Dashboard: "Your adventure begins here" + CTA to `/courses` |
| Zero streak | Grey flame icon, "0 days" text |
| No badges earned | All badges as locked silhouettes with names visible |
| Long TC text in cards | `overflow-wrap: break-word`, no mid-character truncation |
| 375px quiz options | Stack vertically, full-width, min 44px height touch targets |
| Mobile lesson sidebar | Hamburger icon → slide-out drawer |
| Mobile sticky purchase card | Fixed bottom bar: price + CTA button |
| Exam question 0/60 | HP at 100%, counter shows "1 / 60" |
| Exam time expired | Show result page with current answers (placeholder behavior) |

## Pre-Delivery Checklist (from MASTER.md)

Every page must pass before M0001 is complete:

- [ ] No emojis as icons — Lucide SVG only
- [ ] All colors from design system palette
- [ ] Font pairing: Poppins headings, Open Sans body, Noto Sans TC for CJK
- [ ] Hover states don't cause layout shift
- [ ] All clickable elements have `cursor-pointer`
- [ ] Transitions 150-300ms
- [ ] Focus states visible for keyboard navigation
- [ ] Text contrast >= 4.5:1
- [ ] Borders visible on dark background
- [ ] RPG accents use `-400` weight
- [ ] Responsive at 375px, 768px, 1024px, 1440px
- [ ] No horizontal scroll on mobile
- [ ] No content hidden behind fixed navbars
- [ ] Noto Sans TC loaded and rendering correctly
- [ ] Line height >= 1.7 for CJK body text
- [ ] All UI copy in 繁體中文
- [ ] RPG elements use accent colors, not main palette
- [ ] No cartoon/childish visual treatments
