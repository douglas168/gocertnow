**Date:** 2026-03-31
**Milestone:** M0001-20260331-web-rpg-lms-shell

## Overview

Build the complete LevelCert visual shell: 8 pages in Next.js 16 with the Indigo + Emerald dark-mode-only design system, shadcn/ui components, RPG-lite gamification elements, and hardcoded Traditional Chinese placeholder data. Work is split into 6 phases — foundation cleanup, shared components, marketing pages, app pages, quiz/exam pages, and polish. Each phase must build and render before moving to the next.

| Package | Version | Notes |
|---------|---------|-------|
| Next.js | 16.2.x (current) | Turbopack + React Compiler |
| Tailwind CSS | 4.2.x | Upgrade from current v4 if needed |

| React | 19.x (current) | Required by Next.js 16 |

### Task 0.4 — Replace globals.css with design system tokens

**File:** `web/app/globals.css`

Replace entire file. Key changes:
- Remove light mode variables, `.dark` class, `@theme inline`
- Use `@theme` (without `inline`) for Tailwind v4 token registration
- Define all MASTER.md colors as CSS variables in `:root`
- Register font families, spacing, shadows, border-radius as Tailwind theme tokens
- Set `body` styles: `bg-background`, `text-text`, `font-body`, `line-height: 1.75`

**CSS variable structure:**
```
:root {
  /* Core palette — from MASTER.md */
  --color-primary, --color-secondary, --color-cta
  --color-background, --color-surface, --color-text, --color-text-muted, --color-border

  /* RPG accents */
  --color-xp, --color-streak, --color-success, --color-danger, --color-badge, --color-radar

  /* Exam overrides */
  --color-exam-bg, --color-exam-surface

  /* Spacing, shadows, radius — per MASTER.md tables */
}

@theme {
  --font-heading: var(--font-heading), var(--font-cjk), system-ui, sans-serif;
  --font-body: var(--font-body), var(--font-cjk), system-ui, sans-serif;
  /* Map CSS vars to Tailwind tokens */
}
```

**Gotchas:**
- Tailwind v4 `@theme` syntax differs from v3 `tailwind.config.ts`. Consult Tailwind v4 docs.
- shadcn/ui expects certain CSS variable names (`--primary`, `--secondary`, `--background`, etc.). Verify shadcn's generated `globals.css` section and merge with our design tokens — don't fight the convention, map to it.
- Test that `bg-primary`, `text-primary`, `bg-surface` etc. resolve correctly in a test component.

### Task 4.2 — Section quiz (`/learn/[slug]/quiz/[section]`)

**File:** `web/app/(app)/learn/[slug]/quiz/[section]/page.tsx`

**Note:** This lives in the `(app)` route group (uses app navbar), NOT in `(exam)`.

This page needs client-side interactivity — `"use client"`.

**State management:**
- `currentQuestion: number` (0-indexed)
- `selectedAnswer: number | null`
- `showFeedback: boolean`
- `answers: { selected: number, correct: boolean }[]`
- `quizComplete: boolean`

**Flow:**
1. Intro screen: "小怪戰鬥: [Section Name]" — "5 題, 無時間限制, 可重複挑戰"
2. Question display: `<QuestionCard>` with 4 options
3. Click option → set `selectedAnswer`
4. Click "確認答案" → set `showFeedback`, show `<AnswerFeedback>`
5. Click "下一題" → advance `currentQuestion`
6. After last question → completion screen: score (X/5), XP earned, badge unlock if applicable
7. CTAs: "重新挑戰" (outline) | "繼續下一章" (emerald)

**Progress:** Dots at top — filled = answered

**Max width:** `max-w-2xl`
**Match:** `docs/product-mockups/04-section-quiz.html`

### Task 4.3 — Mock exam (`/exam/[slug]`)

**File:** `web/app/(exam)/exam/[slug]/page.tsx`

`"use client"` — most interactive page in the app.

**State management:**
- `currentQuestion: number`
- `answers: (number | null)[]` (60 slots, null = unanswered)
- `flagged: Set<number>`
- `timeRemaining: number` (seconds, countdown)
- `examSubmitted: boolean`

**Components on page:**
- **Top bar (fixed):**
  - `<HpBar>` — `current` = total correct so far, `max` = total answered
  - Timer — `font-mono`, `text-lg`. Turns red-400 at < 300s (5 min). Use `setInterval` for countdown.
  - Question counter: "23 / 60"
  - "Exit" ghost button → confirmation dialog (shadcn `AlertDialog`)

- **Question area:**
  - Category tag pill above question
  - Question text + 4 option cards
  - No immediate feedback (unlike section quiz)

- **Navigation:**
  - "Previous" (ghost) / "Next" (emerald) buttons
  - Question grid: clickable numbered circles. States: answered (filled indigo), current (ring), unanswered (outline), flagged (amber fill)
  - "Flag for review" toggle per question

- **Submit:**
  - After Q60 or manual: confirmation dialog showing unanswered + flagged counts
  - "提交考試" CTA → navigate to result page

**Transitions:** Question fade (150ms opacity), HP bar width (300ms)

**Max width:** `max-w-3xl`
**Match:** `docs/product-mockups/06-exam.html`

**Gotchas:**
- Timer must use `useEffect` with cleanup to avoid memory leaks
- Question grid on mobile: horizontal scroll strip with `overflow-x-auto`
- HP bar gradient trick: fixed-width gradient div inside `overflow-hidden` wrapper that shrinks
- Use `sampleExam` data (10 questions for placeholder — UI shows "10/10" not "60/60" since we only have 10 sample questions)
- On submit: store answers in state/URL params to pass to result page. Since no backend, use `sessionStorage` or route with query params.
